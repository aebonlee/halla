왜 아이디어가 먼저인가

좋은 생성형 AI 서비스는 화려한 기술이 아니라 "누구의 어떤 문제를 푸는가"에서 출발합니다. 기술(LangChain)은 그 문제를 푸는 도구일 뿐입니다.

그래서 코딩보다 먼저 ① 디자인 씽킹(사람의 문제 발견)과 ② 컴퓨터 사고력(문제를 풀 수 있게 쪼개기)을 거칩니다.

💡 순서
문제 발견(디자인 씽킹) → 문제 구조화(컴퓨터 사고력) → 한 줄 기획 → LangChain 구성 선택 → 구현.

디자인 씽킹 5단계

스탠퍼드 d.school의 디자인 씽킹은 "사람 중심"으로 문제를 찾고 해결하는 5단계 과정입니다. 각 단계를 생성형 AI 프로젝트에 적용해 봅니다.

① 공감 (Empathize)

  • 대상 사용자를 관찰·인터뷰해 진짜 불편을 찾는다
  • AI 적용: "신입생은 학칙을 어디서 찾는지 몰라 매번 사무실에 전화한다"
  • 도구: 사용자 인터뷰, 관찰 일지, 공감 지도

② 정의 (Define)

  • 관찰을 한 문장의 문제 정의로 좁힌다 (POV)
  • 예: "신입생은 학칙 정보를 빠르게 찾고 싶다. 왜냐하면 전화·검색이 번거롭기 때문이다."

③ 아이디어 (Ideate)

  • 판단 없이 많은 해결책을 쏟아낸다(브레인스토밍)
  • 예: 챗봇 / FAQ 자동분류 / 학칙 요약 카드 / 음성 안내 …
  • 도구: 브레인스토밍, 크레이지8, How Might We 질문

④ 프로토타입 (Prototype)

  • 가장 작고 빠른 버전을 만든다 (완벽 X)
  • AI 적용: 학칙 PDF 3장만 넣은 RAG 챗봇으로 핵심 동작만 확인

⑤ 테스트 (Test)

  • 실제 사용자에게 써보게 하고 피드백으로 개선
  • AI 적용: 신입생 3명에게 질문시켜 "못 찾는 질문" 수집 → 문서·프롬프트 보강
📝 핵심
디자인 씽킹은 직선이 아니라 반복입니다. 테스트에서 배운 걸 다시 정의·아이디어로 되돌립니다.

컴퓨터 사고력 4요소

컴퓨팅 사고력(Computational Thinking)은 문제를 컴퓨터(그리고 AI)가 풀 수 있는 형태로 바꾸는 사고법입니다.

① 분해 (Decomposition)

  • 큰 문제를 작은 문제로 쪼갠다
  • AI 적용: "학칙 챗봇" = 문서 수집 → 분할 → 임베딩 → 검색 → 답변 → 화면

② 패턴 인식 (Pattern Recognition)

  • 반복되는 구조·유형을 찾는다
  • AI 적용: 들어오는 질문이 "기한/장소/방법" 유형으로 반복됨 → 분류기 도입

③ 추상화 (Abstraction)

  • 불필요한 세부는 버리고 핵심만 남긴다
  • AI 적용: 모든 문서를 "청크 + 벡터"라는 한 가지 형태로 추상화

④ 알고리즘 (Algorithm)

  • 해결을 단계(순서)로 정의한다
  • AI 적용 (RAG): 질문 → 임베딩 → 유사 청크 검색 → 프롬프트 삽입 → 생성 → 출력
💡 연결
디자인 씽킹이 "무엇을/왜"라면, 컴퓨터 사고력은 "어떻게 풀 수 있게 쪼갤까"입니다. 둘을 거치면 코드로 옮기기 쉬워집니다.

아이디어 발상 5단계

  1. 문제 발견 — 내 주변(학과·동아리·아르바이트)의 반복되는 불편 5개 적기
  2. 사용자 정의 — 그 불편을 겪는 사람을 한 명으로 구체화 (페르소나)
  3. 브레인스토밍 — 해결책을 10개 이상 막 쏟아내기 (질·판단 금지)
  4. 선별 — 임팩트(도움 정도) × 실현가능성(6일 내 가능?)으로 점수 매겨 1개 선택
  5. 한 줄 기획 — "누구의 / 어떤 문제를 / 무엇으로 푼다"로 압축 + 범위 좁히기
📝 선별 팁
"멋지지만 6일 안에 못 만드는 것"보다 "작지만 끝까지 작동하는 것"을 고르세요.

한 장 기획 워크시트

아래 칸을 채우면 바로 설계로 넘어갈 수 있습니다.

[프로젝트 한 장 기획서]

1) 누구를 위한 것인가 (사용자/페르소나):
2) 어떤 문제/불편을 푸는가:
3) 핵심 기능 1개 (가장 중요한 것 하나):
4) 사용할 데이터/문서:
5) LangChain 구성 (RAG / 챗봇 / 분류 / 요약 중):
6) 데모 시나리오 (시연할 질문 3개):
7) 성공 기준 (무엇이 되면 성공인가):
8) 6일 내 안 할 것 (범위 밖):
📝 체크
7번(성공 기준)과 8번(범위 밖)을 적으면 "작게 완성"이 쉬워집니다.
