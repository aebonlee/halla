학습 목표

  • 좋은 프롬프트의 4원칙을 적용하고 템플릿으로 재사용할 수 있다.
  • few-shot 예시로 출력 형식을 안정화할 수 있다.

  • 커스텀 OutputParser로 모델 출력을 가공하고 외부 함수(numexpr)와 연결할 수 있다.

프롬프트 작성 원칙

프롬프트는 사용자와 모델 사이의 "요청서"입니다. 잘 쓰면 환각이 줄고 원하는 형식이 안정됩니다.

  • 명확성·구체성 — 무엇을, 어떤 형식으로 원하는지 분명히
  • 맥락 제공 — 배경 정보를 주면 관련성↑, 환각↓
  • 간결함 — 핵심에 집중, 불필요한 정보 배제
  • 열린 질문 + 예시 — 원하는 답의 예를 1~2개 보여주면(few-shot) 일관성↑
📝 한 줄
"역할 + 맥락 + 출력형식 + 예시" 네 가지를 갖추면 대부분의 프롬프트가 좋아집니다.

PromptTemplate 기초

변수 {}가 들어간 틀을 만들어 값만 바꿔 끼웁니다. 단일 문장용은 PromptTemplate.

from langchain_core.prompts import PromptTemplate

tmpl = PromptTemplate.from_template("{name}님은 {age}살입니다. {language}로 인사해줘")

# format(): 값을 채워 "문자열"로 확인
print(tmpl.format(name="홍길동", age=20, language="영어"))
# → 홍길동님은 20살입니다. 영어로 인사해줘

템플릿 조합 (+)

템플릿끼리 +로 이어 붙여 "기본 틀 + 규칙"을 합칠 수 있습니다.

from langchain_core.prompts import PromptTemplate

base = PromptTemplate.from_template("{topic}에 대해 설명해줘")
rule = PromptTemplate.from_template("\n\n조건: 3문장 이내, 중학생 눈높이로")
combined = base + rule           # 두 틀을 합침

print(combined.format(topic="블랙홀"))

few-shot 예시

"이렇게 답해줘"라는 예시를 프롬프트에 넣으면 형식이 안정됩니다.

from langchain_core.prompts import ChatPromptTemplate
from langchain_openai import ChatOpenAI

# 예시 2개를 보여주고, 같은 형식으로 답하게 유도(few-shot)
prompt = ChatPromptTemplate.from_messages([
    ("system", "너는 단어를 받아 '단어: 한줄뜻' 형식으로만 답한다."),
    ("human", "사과"),   ("ai", "사과: 빨갛고 단 과일"),
    ("human", "고래"),   ("ai", "고래: 바다에 사는 큰 포유류"),
    ("human", "{word}"),                       # ← 실제 입력
])
chain = prompt | ChatOpenAI(temperature=0)
print(chain.invoke({"word": "은하"}).content)

ChatPromptTemplate

대화형은 ChatPromptTemplate.from_messages로 system/human 메시지를 함께 구성합니다. 각 메시지가 변수를 가질 수 있습니다.

from langchain_core.prompts import ChatPromptTemplate

chat_prompt = ChatPromptTemplate.from_messages([
    ("system", "You are a helpful assistant that translates {input_language} to {output_language}."),
    ("human",  "{user_input}"),
])
msgs = chat_prompt.format_messages(
    input_language="English", output_language="Korean", user_input="I love programming")
print(msgs)   # SystemMessage(...), HumanMessage(...) 로 채워진 리스트

LCEL — 파이프 |

LangChain의 정수는 표현언어(LCEL) — | 파이프로 구성요소를 잇는 것입니다. 데이터가 왼쪽→오른쪽으로 흐릅니다.

from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser
from langchain_openai import ChatOpenAI

prompt = ChatPromptTemplate.from_template("Tell me a short joke about {topic}")
chat_model = ChatOpenAI()
output_parser = StrOutputParser()

# 체인 "정의" (아직 호출 안 함) — 프롬프트 → 모델 → 파서
chain = prompt | chat_model | output_parser

# invoke에 변수는 dict로 전달 (프롬프트가 dict를 받기 때문)
print(chain.invoke({"topic": "ice cream"}))

StrOutputParser

모델 응답은 메시지 객체입니다. StrOutputParser를 체인 끝에 붙이면 문자열만 깔끔히 나옵니다(.content 불필요).

# 객체를 따로 안 만들고 한 줄로도 가능
chain = prompt | ChatOpenAI() | StrOutputParser()
print(chain.invoke({"topic": "ice cream"}))
💡 파서의 역할
모델 출력을 "프로그램이 쓰기 좋은 형태"로 변환. 문자열(StrOutputParser), JSON, 정수 등 목적에 맞게.

LCEL 원리 (__or__)

|는 마법이 아니라 파이썬 OR 연산자 오버로딩입니다. 작은 클래스를 직접 만들어 원리를 체험합니다 (lang_02 원본).

class Runnable:
    def __init__(self, func):
        self.func = func

    # a | b 를 만나면 호출됨 → 두 함수를 이어 붙임
    def __or__(self, other):
        def chained_func(*args, **kwargs):
            # self의 결과를 other에 넘김 (왼→오 흐름)
            return other(self.func(*args, **kwargs))
        return Runnable(chained_func)

    def __call__(self, *args, **kwargs):
        return self.func(*args, **kwargs)

함수 체이닝 예제

일반 함수도 위 Runnable로 감싸면 |로 이을 수 있습니다.

def add_five(x):  return x + 5
def multiply_by_two(x): return x * 2
def num2ko(x): return str(x) + "입니다."

add_five = Runnable(add_five)
multiply_by_two = Runnable(multiply_by_two)
num2ko = Runnable(num2ko)

chain = add_five | multiply_by_two | num2ko
print(chain(3))    # (3+5)*2 = 16 → "16입니다."
📝 깨달음
LCEL의 prompt | model | parser도 결국 "앞의 결과를 뒤로 넘기는" 같은 원리입니다.

RunnablePassthrough

체인은 보통 dict를 받습니다. RunnablePassthrough를 쓰면 값 하나만 넣어도 동작하게 만들 수 있습니다.

from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser
from langchain_openai import ChatOpenAI
from langchain_core.runnables import RunnablePassthrough

prompt = ChatPromptTemplate.from_template("Tell me a short joke about {topic}")
chain = {"topic": RunnablePassthrough()} | prompt | ChatOpenAI() | StrOutputParser()

# "ice cream" → {"topic": "ice cream"} 으로 자동 포장되어 흐름
print(chain.invoke("ice cream"))

assign으로 변수 자동 채우기

변수가 여러 개인데 사용자는 하나만 입력하게 하고 싶을 때, RunnablePassthrough.assign으로 나머지를 함수가 자동으로 채웁니다 (lang_02 원본: 대화 예문 생성기).

from langchain_core.runnables import RunnablePassthrough

template = "Please generate three dialogue sentences in {language} on {topic} for a {level} level"
prompt = ChatPromptTemplate.from_template(template)

# 시스템이 정하는 값(language/level)을 함수로 제공 — 인자는 1개(_)만 받는다
def get_language(_): return "English"
def get_level(_):    return "beginner"

chain = (
    RunnablePassthrough.assign(language=get_language, level=get_level)
    | prompt | ChatOpenAI() | StrOutputParser()
)
# 사용자는 topic 하나만 입력 → language/level은 자동으로 채워짐
print(chain.invoke({"topic": "travel"}))

Parallel · Lambda

RunnableParallel은 여러 작업을 동시에, RunnableLambda는 임의의 함수를 체인에 끼웁니다.

from langchain_core.runnables import RunnableParallel, RunnableLambda

# 한 입력으로 두 결과를 동시에 만들기
joke  = ChatPromptTemplate.from_template("{t} 농담") | ChatOpenAI() | StrOutputParser()
fact  = ChatPromptTemplate.from_template("{t} 사실 1개") | ChatOpenAI() | StrOutputParser()
both  = RunnableParallel(joke=joke, fact=fact)
print(both.invoke({"t": "고양이"}))   # {"joke": "...", "fact": "..."}

# 중간에 함수 끼우기(대문자로)
upper = RunnableLambda(lambda s: s.upper())
chain = ChatPromptTemplate.from_template("{t} in english, one word") | ChatOpenAI() | StrOutputParser() | upper
print(chain.invoke({"t": "사과"}))

커스텀 OutputParser

모델 답에서 원하는 부분만 뽑도록 파서를 직접 만들 수 있습니다 (lang_03 원본).

from langchain_core.output_parsers import BaseOutputParser

class CodeOutputParser(BaseOutputParser):
    def parse(self, text: str):
        # 모델이 ```text\n15**0.3432\n``` 처럼 답하면 가운데 수식만 추출
        value = text.strip().split("```")
        return value[1].split("\n")[1]

수학 체인 (실전)

자연어 질문 → LLM이 수식 생성 → 파서로 추출 → numexpr로 실제 계산. 여러 블록을 이어 "하는 일"을 완성합니다.

import numexpr
numexpr.evaluate("15**0.3432")   # 파이썬으로 수식 계산 확인
from langchain_core.prompts import ChatPromptTemplate
from langchain_openai import ChatOpenAI
from langchain_core.runnables import RunnablePassthrough

system = """You make ONLY an argument for numexpr.evaluate from the user query.
Respond as a code block only, like:
```text
13**0.3432
```
without any other description."""

prompt = ChatPromptTemplate.from_messages([("system", system), ("human", "{user_input}")])

# 질문 → 수식 생성 → 수식 추출 → numexpr로 계산까지 한 줄 체인
chain = ({"user_input": RunnablePassthrough()}
         | prompt | ChatOpenAI(temperature=0) | CodeOutputParser() | numexpr.evaluate)
print(chain.invoke("What is 15 raised to the .3432 power?"))

확장 — 단위 변환기

# 같은 패턴으로 "단위 변환"도 가능 (자연어 → 수식 → 계산)
chain2 = ({"user_input": RunnablePassthrough()}
          | ChatPromptTemplate.from_messages([
                ("system", system),
                ("human", "{user_input}")])
          | ChatOpenAI(temperature=0) | CodeOutputParser() | numexpr.evaluate)
print(chain2.invoke("5 km is how many meters?"))   # 5000 류

  1. few-shot 예시 2개로 "단어 → 한줄뜻" 형식을 고정하는 체인 만들기
  2. LCEL prompt | model | StrOutputParser() 체인 1개 완성
  3. __or__ Runnable로 숫자 함수 3개를 이어 결과 확인
  4. assign으로 사용자는 topic만 입력하는 "대화 예문 생성기" 만들기
  5. 수학 체인을 변형해 "온도(°F↔°C) 변환기" 만들기

자주 막히는 부분

  • KeyError: 'topic' → invoke에 dict로 변수를 안 넣었거나 키 이름이 다름
  • 출력이 메시지 객체로 나옴 → 체인 끝에 StrOutputParser() 추가
  • 파서 IndexError → 모델이 코드블록(```) 형식을 안 지킴. system 프롬프트를 더 강하게
  • numexpr 미설치 → !pip install numexpr
