학습 목표

  • "AI에게 시켜서 만든다"는 바이브 코딩 방식으로 개발을 시작할 수 있다.
  • Colab + OpenAI API 키 환경을 세팅하고 LangChain을 설치할 수 있다.
  • LangChain으로 LLM을 호출하고, 프롬프트 템플릿으로 첫 LLM 체인을 만들 수 있다.

바이브 코딩으로 시작

이 과정은 코드를 외워서 치는 게 아니라, AI에게 자연어로 시켜서 만드는 바이브 코딩으로 시작합니다. 막히면 Claude·Cursor에게 코드를 설명·수정·디버깅하도록 요청하세요.

📝 오늘의 마음가짐
"이 코드가 무슨 뜻이야?", "이 에러 고쳐줘", "이걸 ○○하게 바꿔줘" — 정확히 묘사하는 능력이 새로운 코딩 실력입니다.

개발 환경 — Colab + API 키

설치 없이 브라우저에서 바로 쓰는 Google Colab으로 실습합니다. 첫 셀에서 LangChain을 설치합니다.

!pip install langchain>=0.3.0 langchain-openai langchain-community langchain-text-splitters
!pip install faiss-cpu tiktoken   # 벡터 검색·토큰 계산(이후 사용)

import langchain
print("LangChain 버전 :", langchain.__version__)

API 키는 코드에 직접 쓰지 않고 Colab 보안 비밀(🔑)에 저장한 뒤 불러옵니다.

import os
from google.colab import userdata
os.environ["OPENAI_API_KEY"] = userdata.get("OPENAI_API_KEY")
⚠️ 키 보안
API 키는 절대 공개 코드/깃허브에 올리지 않습니다. 노출 시 요금 폭탄·오남용 위험이 있습니다.

첫 LLM 호출

ChatOpenAI 모델을 만들고 invoke()로 질문하면 끝입니다.

from langchain_openai import ChatOpenAI

llm = ChatOpenAI(model="gpt-4o-mini")
response = llm.invoke("지구의 윤년 주기는?")
print(response.content)
💡 temperature
ChatOpenAI(model="gpt-4o-mini", temperature=0) 은 일관된 답, temperature=1 은 창의적이고 매번 달라지는 답을 줍니다. 사실 질문은 낮게, 아이디어는 높게.

메시지 & 프롬프트 템플릿

대화는 역할(role)이 있는 메시지로 구성됩니다 — system(규칙)·human(사용자)·ai(응답).

from langchain_core.messages import SystemMessage, HumanMessage

messages = [
    SystemMessage(content="너는 한라대 학생을 돕는 친절한 학습 도우미야."),
    HumanMessage(content="오늘 배운 LangChain을 3줄로 요약해줘"),
]
print(llm.invoke(messages).content)

반복되는 프롬프트는 템플릿으로 만들어 값만 바꿔 끼웁니다.

from langchain_core.prompts import ChatPromptTemplate

prompt = ChatPromptTemplate.from_template("{topic}에 대해 한 문장으로 설명해줘")
print(prompt.format(topic="머신러닝"))

LLM 체인 (Prompt + LLM)

LangChain의 핵심은 체인 — 프롬프트와 모델을 이어 "입력 → 처리 → 출력"의 흐름을 만드는 것입니다. (내일 배울 LCEL의 맛보기)

chain = prompt | llm          # 프롬프트 → 모델 연결
result = chain.invoke({"topic": "딥러닝"})
print(result.content)
🔭 오늘의 한 줄
LangChain = LLM 애플리케이션을 "블록 조립"처럼 만드는 프레임워크. 오늘은 블록 1개(LLM 체인)를 만들었습니다.

오늘의 과제

  1. Colab에서 첫 LLM 호출 성공 + 스크린샷
  2. 나만의 system 프롬프트(역할·말투·규칙)로 도우미 1개 만들기
  3. temperature 0 / 0.7 / 1.0 으로 같은 질문 호출해 차이 비교
