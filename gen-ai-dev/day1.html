학습 목표

  • "AI에게 시켜서 만든다"는 바이브 코딩 방식으로 개발을 시작할 수 있다.
  • LangChain의 4대 구성요소와 "체인" 개념을 설명할 수 있다.
  • Colab에서 API 키를 안전하게 다루고 LangChain을 설치·검증할 수 있다.
  • LLM 호출 → 응답 객체 해석 → 메시지 구성 → 프롬프트 템플릿 → 체인까지 직접 만들 수 있다.
  • temperature·max_tokens·스트리밍·토큰 사용량의 의미를 이해하고 조절할 수 있다.

바이브 코딩으로 시작

이 과정은 코드를 통째로 외우는 게 아니라 AI에게 자연어로 시켜서 만들고 사람이 이해·수정하는 "바이브 코딩"으로 출발합니다.

  • "이 코드가 무슨 뜻인지 한 줄씩 설명해줘"
  • "이 에러 메시지를 보고 원인과 해결책을 알려줘"
  • "이 코드를 ○○하게 바꿔줘. 바뀐 부분만 설명해줘"
📝 오늘의 마음가짐
문법을 못 외워도 됩니다. "원하는 것을 정확히 묘사하는 능력"이 새로운 코딩 실력입니다.

LangChain 4대 구성요소

LangChain은 LLM으로 앱을 만드는 프레임워크입니다. "AI 블록"을 조립한다고 생각하세요.

  • LangChain 라이브러리 — 프롬프트·모델·파서를 체인으로 결합
  • LangChain 템플릿 — 자주 쓰는 작업의 배포용 아키텍처 모음
  • LangServe — 만든 체인을 REST API로 배포(Day 5)
  • LangSmith — 체인 디버깅·테스트·평가·모니터링
💡 "체인"이란?
입력(프롬프트)을 받아 LLM으로 결과를 만드는 흐름. 가장 기본은 프롬프트 + LLM = LLM 체인.

개발 환경 · API 키

설치 없이 브라우저에서 쓰는 Google Colab으로 실습합니다. 한 줄씩 의미를 보며 따라오세요.

# !pip : Colab(주피터)에서 "파이썬 패키지를 설치"하는 명령. 맨 앞 ! 는 "터미널 명령"이라는 뜻.
# langchain          : 랭체인 본체
# langchain-openai   : 랭체인 ↔ OpenAI 연결 도구(ChatOpenAI 등)
# langchain-community: 커뮤니티가 만든 추가 도구(문서 로더 등, 뒤에서 사용)
# langchain-text-splitters : 긴 문서를 자르는 도구(Day4~5에서 사용)
!pip install langchain>=0.3.0 langchain-openai langchain-community langchain-text-splitters

# faiss-cpu : 벡터 검색 라이브러리(대안 — 이 과정의 Day4~5는 Chroma 사용) / tiktoken : 토큰 개수 계산기
!pip install faiss-cpu tiktoken

# 설치가 잘 됐는지 "버전"을 찍어 확인 (숫자가 나오면 성공)
import langchain                     # 방금 설치한 랭체인을 불러옴(import)
print(f"LangChain 버전 : {langchain.__version__}")   # __version__ : 설치된 버전 문자열

API 키를 "보안 비밀"에 저장하기

키를 코드에 직접 쓰면 새어 나갑니다. Colab 왼쪽 🔑(보안 비밀)에 등록한 뒤 코드로 불러옵니다.

import os                              # os : 운영체제·환경변수를 다루는 표준 도구
from google.colab import userdata      # userdata : Colab의 "보안 비밀"을 읽는 도구

# userdata.get("이름") : 보안 비밀에 저장해 둔 값을 가져옴
# os.environ["OPENAI_API_KEY"] = ... : 그 키를 "환경변수"에 넣어 둠
#   → 이렇게 넣어두면 ChatOpenAI() 가 자동으로 이 키를 찾아 씁니다(코드에 키를 안 써도 됨)
os.environ["OPENAI_API_KEY"] = userdata.get("OPENAI_API_KEY")

# 키 값 자체는 절대 print 하지 말 것! 들어왔는지 여부(True/False)만 확인
print("키 등록됨:", bool(os.environ.get("OPENAI_API_KEY")))
⚠️ 키 보안
API 키는 공개 코드·깃허브·스크린샷에 절대 올리지 않습니다. 노출되면 타인이 내 요금으로 쓸 수 있어요. "키 = 돈".

첫 LLM 호출

모델 객체를 만들고 invoke()로 질문하면 끝입니다. 두 종류 모델을 비교해 봅니다.

① 일반 LLM — 문자열 in, 문자열 out (엽서)

from langchain_community.llms import OpenAI   # 옛 방식 모델(completion)

llm = OpenAI()                  # 모델 객체 생성 (키는 환경변수에서 자동으로 가져감)
prompt = "안녕하세요"            # 모델에게 보낼 입력(그냥 문자열)
response = llm.invoke(prompt)   # invoke : "한 번 호출해서 답을 받아라"
print(response)                 # 출력도 그냥 문자열
# 흐름:  문자열 입력  →  LLM  →  문자열 출력

② 채팅 모델 — 메시지 in, 메시지 out (편지봉투+편지지) · 권장

from langchain_openai import ChatOpenAI       # 요즘 표준(chat) 모델

chat = ChatOpenAI(model="gpt-4o-mini")          # 모델 지정: gpt-4o-mini(빠르고 저렴)
response = chat.invoke("지구의 윤년 주기는?")     # 문자열만 넣어도 알아서 메시지로 감쌈

print(response)             # ← AIMessage(content='...') 라는 "객체" 전체가 찍힘
print()                     # 빈 줄 한 칸
print(response.content)     # ← .content : 그 객체 안의 "답변 텍스트"만 꺼냄
💡 LLM vs ChatModel
LLM은 "엽서"(문자열→문자열), ChatModel은 "편지봉투+편지지"(역할 있는 메시지). 앞으로 거의 ChatModel(ChatOpenAI)만 씁니다.

확장 — 여러 질문을 반복해서 물어보기

# 질문들을 리스트(목록)에 담아두고, for 반복문으로 하나씩 물어봄
questions = [
    "파이썬의 장점 한 가지만 말해줘",
    "리스트와 튜플의 차이를 한 문장으로",
]
for q in questions:                 # questions 안의 항목을 q에 하나씩 꺼내 반복
    answer = chat.invoke(q).content # 호출 후 바로 .content 로 텍스트만 받기
    print(f"Q: {q}")                # f"...{변수}..." : 문자열 안에 변수 값을 끼워넣기(f-string)
    print(f"A: {answer}\n")        # \n : 줄바꿈 한 줄

응답 객체 뜯어보기

chat.invoke()의 결과는 단순 문자열이 아니라 AIMessage 객체입니다. 안을 들여다보면 토큰 사용량까지 보입니다.

resp = chat.invoke("한국의 수도는?")

print(type(resp))             # type() : 자료형 확인 → AIMessage 객체임을 알 수 있음
print(resp.content)           # 답변 텍스트
print(resp.response_metadata) # 모델명·종료이유 등 부가정보(딕셔너리 형태)
print(resp.usage_metadata)    # 토큰 사용량 {입력, 출력, 합계}

# usage_metadata 는 딕셔너리(dict). 딕셔너리는 ["키"] 로 값을 꺼냄
u = resp.usage_metadata
print(f"입력 {u['input_tokens']} + 출력 {u['output_tokens']} = 총 {u['total_tokens']} 토큰")
📝 왜 보나요?
토큰 = 비용입니다. 응답에서 사용량을 바로 확인하는 습관을 들이면 비용 큰 호출을 미리 잡습니다.

메시지 구조 (편지봉투)

채팅은 역할(role)이 있는 메시지들의 리스트입니다. "편지봉투(messages) 안에 편지지(각 메시지)를 담는다"고 생각하세요.

  • SystemMessage — 개발자가 정하는 규칙/성격(대화 내내 유지)
  • HumanMessage — 사용자가 보낸 말
  • AIMessage — AI가 답한 말
from langchain_openai import ChatOpenAI
from langchain_core.messages import SystemMessage, HumanMessage   # 메시지 종류들

chat = ChatOpenAI(temperature=0)        # temperature=0 : 흔들림 없이 일관된 답

# messages 는 "리스트"([])이고, 그 안에 메시지 객체를 순서대로 담음
messages = [
    SystemMessage(content="너는 영어를 한국어로 번역하는 번역가야."),  # 규칙(역할) 설정
    HumanMessage(content="I love programming"),                    # 사용자가 보낸 문장
]
print(chat.invoke(messages).content)    # 메시지 리스트를 통째로 모델에 전달
💡 핵심
같은 모델도 SystemMessage만 바꾸면 완전히 다른 비서가 됩니다. "역할 지정"이 품질의 8할.

번역기 만들기 (원본 + 확장)

System 메시지로 "번역가" 역할을 고정하면 번역기가 됩니다. 원본 → 함수화 → 말투 지정 순으로 발전.

원본 — 영어를 한국어로

from langchain_openai import ChatOpenAI
from langchain_core.messages import SystemMessage, HumanMessage

chat = ChatOpenAI(temperature=0)        # 번역은 일관성이 중요 → 0
messages = [
    SystemMessage(content="너는 번역가야. 사용자의 문장을 한국어로만 번역해."),  # 역할 고정
    HumanMessage(content="Practice makes perfect."),                          # 번역할 문장
]
print(chat.invoke(messages).content)

확장 ① — 도착 언어를 함수의 입력값으로

# def 함수이름(입력값): ... return 결과   → 같은 작업을 입력만 바꿔 재사용
def translate(text, target="Korean"):     # target 기본값은 "Korean"
    """text를 target 언어로 번역해 문자열로 돌려준다."""   # 함수 설명(독스트링)
    msgs = [
        SystemMessage(content=f"너는 번역가야. {target}로만 번역하고 다른 말은 하지 마."),
        HumanMessage(content=text),
    ]
    return chat.invoke(msgs).content      # 결과 텍스트를 돌려줌(return)

print(translate("Good morning", "Japanese"))  # 일본어로 번역
print(translate("좋은 아침이에요", "English"))   # 한국어 → 영어

확장 ② — 말투까지 지정

def translate_tone(text, target="Korean", tone="공손한 존댓말"):
    msgs = [
        # f-string 으로 target, tone 두 값을 규칙 문장에 끼워넣음
        SystemMessage(content=f"{target}로 번역해. 말투: {tone}. 번역문만 출력."),
        HumanMessage(content=text),
    ]
    return chat.invoke(msgs).content

print(translate_tone("Where is the station?", "Korean", "친구에게 하는 반말"))
print(translate_tone("Where is the station?", "Korean", "정중한 비즈니스 존댓말"))

temperature 심화

temperature는 답변의 "무작위성/창의성"을 조절합니다. 0=일관, 1=다양. 직접 비교합니다.

from langchain_openai import ChatOpenAI

# 같은 질문을 temperature 만 다르게 4번 호출 (0 두 개, 1 두 개)
models = [
    ChatOpenAI(model="gpt-4o-mini", temperature=0, max_tokens=120),  # 일관적
    ChatOpenAI(model="gpt-4o-mini", temperature=0, max_tokens=120),
    ChatOpenAI(model="gpt-4o-mini", temperature=1, max_tokens=120),  # 창의적
    ChatOpenAI(model="gpt-4o-mini", temperature=1, max_tokens=120),
]
for i, m in enumerate(models):          # enumerate : 번호(i)와 항목(m)을 함께 꺼냄
    print(f"--- {i} (temp={m.temperature}) ---")
    print(m.invoke("왜 파이썬이 인기 있어? 한 문장으로.").content)
📝 언제 뭘 쓰나
사실·번역·분류 → 0(재현 가능). 아이디어·카피 → 0.7~1.0(다양성).

max_tokens 등 파라미터

모델을 만들 때 동작을 바꾸는 주요 옵션들입니다.

  • model — 모델 선택(gpt-4o-mini 빠름·저렴 / gpt-4o 더 똑똑)
  • temperature — 창의성(0~1)
  • max_tokens — 답변 최대 길이(작으면 잘리고, 크면 비용↑)
chat = ChatOpenAI(
    model="gpt-4o-mini",   # 어떤 모델을 쓸지
    temperature=0.3,       # 약간의 다양성
    max_tokens=256,        # 답변을 256토큰까지만 생성
)
ans = chat.invoke("LangChain을 처음 배우는 사람에게 3줄로 설명해줘")
print(ans.content)
print("출력 토큰:", ans.usage_metadata["output_tokens"])  # 256 근처면 잘렸을 수도
⚠️ 잘림 주의
답이 문장 중간에 끊기면 max_tokens가 작은 것. 늘리거나 "짧게 답해"라고 지시.

프롬프트 템플릿

매번 같은 문장을 새로 쓰지 않게, 변수 {}가 든 틀을 만들어 값만 바꿔 끼웁니다.

from langchain_core.prompts import ChatPromptTemplate

# {topic} 는 "나중에 채울 빈칸"(변수)
prompt = ChatPromptTemplate.from_template("{topic}에 대해 한 문장으로 설명해줘")

print(prompt.format(topic="머신러닝"))          # format : 빈칸을 채운 "문자열" 확인용
msgs = prompt.format_messages(topic="딥러닝")   # format_messages : 모델에 넣을 "메시지"로
print(ChatOpenAI().invoke(msgs).content)

템플릿 심화·조합

format()은 사람이 볼 문자열, format_messages()는 모델에 넣을 메시지. 호출엔 보통 후자를 씁니다.

확장 — 이메일 작성기 템플릿(변수 여러 개)

from langchain_core.prompts import ChatPromptTemplate
from langchain_openai import ChatOpenAI

email_prompt = ChatPromptTemplate.from_messages([
    ("system", "너는 비즈니스 이메일 작성 도우미야. 정중하고 간결하게 한국어로 써."),
    # \n 은 줄바꿈. {to}{purpose}{tone} 세 개의 빈칸
    ("human", "받는사람: {to}\n목적: {purpose}\n톤: {tone}\n위 내용으로 이메일 초안을 써줘."),
])
msgs = email_prompt.format_messages(   # 빈칸 3개를 한꺼번에 채움
    to="교수님", purpose="과제 기한 3일 연장 요청", tone="아주 공손하게")
print(ChatOpenAI(temperature=0.4).invoke(msgs).content)

LLM 체인 (Prompt | LLM)

프롬프트와 모델을 이어 "입력 → 처리 → 출력" 흐름을 만든 것이 LLM 체인입니다. (내일 배울 LCEL 맛보기 — 파이프 |로 연결)

from langchain_core.prompts import ChatPromptTemplate
from langchain_openai import ChatOpenAI

prompt = ChatPromptTemplate.from_template("다음 문장을 {style} 말투로 바꿔줘: {text}")

# | (파이프): "왼쪽 결과를 오른쪽으로 넘겨라". prompt 결과 → chat 모델로
chain = prompt | ChatOpenAI(model="gpt-4o-mini")

# invoke 에는 빈칸 값을 딕셔너리({"키":"값"})로 전달
out = chain.invoke({"style": "친근한 반말", "text": "회의는 오후 3시에 시작합니다."})
print(out.content)

확장 — 여러 입력을 한 번에 처리(batch)

inputs = [                                   # 처리할 입력들의 목록
    {"style": "정중한 존댓말", "text": "지금 어디야?"},
    {"style": "광고 카피", "text": "오늘 세일합니다."},
]
for r in chain.batch(inputs):     # batch : 목록을 한 번에 처리(invoke 여러 번 대신)
    print("-", r.content)

스트리밍 출력

ChatGPT처럼 답이 한 글자씩 실시간으로 나오게 하려면 stream()을 씁니다.

from langchain_openai import ChatOpenAI
from langchain_core.messages import HumanMessage

llm = ChatOpenAI(model="gpt-4o-mini", temperature=0.7)

# stream() : 답을 "조각(chunk)"으로 나눠서 차례로 돌려줌
for chunk in llm.stream([HumanMessage(content="왜 파이썬이 인기 있어?")]):
    # end="" : 줄바꿈 없이 이어서 출력 / flush=True : 버퍼에 안 모으고 즉시 화면에
    print(chunk.content, end="", flush=True)
💡 원리
전체를 다 만든 뒤 주는 게 아니라, 생성되는 대로 흘려보냅니다(streaming). 기다림이 줄어듭니다.

토큰·콜백 모니터링

실무에선 "얼마나 걸리고 얼마나 쓰는지"를 추적합니다. 콜백 핸들러로 자동 기록할 수 있습니다.

import time                                       # time : 시간 측정 도구
from langchain_core.callbacks.base import BaseCallbackHandler  # 콜백 기본 클래스

# 콜백 핸들러 = "특정 순간(시작/끝)에 자동으로 실행될 코드"를 담는 클래스
class TimerHandler(BaseCallbackHandler):
    def on_llm_start(self, serialized, prompts, **kwargs):   # LLM 호출이 "시작될 때" 자동 실행
        self.t0 = time.time()                                # 시작 시각을 저장
    def on_llm_end(self, response, **kwargs):                # LLM 호출이 "끝날 때" 자동 실행
        print(f"[소요시간] {time.time() - self.t0:.2f}초")    # (끝시각 - 시작시각) = 걸린 시간

# callbacks= 에 핸들러를 넣으면, 호출할 때마다 위 함수들이 자동 실행됨
llm = ChatOpenAI(model="gpt-4o-mini", callbacks=[TimerHandler()])
ans = llm.invoke("LangChain의 장점 3가지")
print(ans.content)
print("총 토큰:", ans.usage_metadata["total_tokens"])
📝 확장
콜백에서 토큰을 누적하면 "이번 세션 총 비용"을 추적할 수 있어요. 모니터링·LangSmith의 출발점.

  2. 나만의 도우미 — system 프롬프트로 역할·말투·금지사항을 정한 비서(예: "초등학생도 알아듣게 설명하는 과학 선생님")

자주 막히는 부분

  • AuthenticationError → 보안 비밀 "이름"과 userdata.get("이름") 불일치, 또는 키 만료
  • RateLimitError → 사용량/결제 한도 초과. 잠시 후 재시도/결제 확인
  • 답이 중간에 잘림 → max_tokens 키우기
  • 한국어가 어색 → system에 "자연스러운 한국어로" 명시
  • ModuleNotFound → 설치 셀 다시 실행 후 런타임 재시작
