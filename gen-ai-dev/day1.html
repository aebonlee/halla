학습 목표

  • "AI에게 시켜서 만든다"는 바이브 코딩 방식으로 개발을 시작할 수 있다.
  • Colab에서 API 키를 안전하게 다루고 LangChain을 설치할 수 있다.
  • LLM 호출 → 메시지 구성 → 프롬프트 템플릿 → 체인까지 직접 만들 수 있다.
  • temperature·스트리밍·토큰 사용량의 의미를 이해한다.

바이브 코딩으로 시작

이 과정은 코드를 외워서 치는 게 아니라 AI에게 자연어로 시켜서 만드는 바이브 코딩으로 출발합니다. 막히면 Claude·Cursor·ChatGPT에게 "이 코드 설명해줘 / 이 에러 고쳐줘 / 이렇게 바꿔줘"라고 요청하세요.

LangChain이란?

대규모 언어 모델(LLM)을 활용해 애플리케이션을 만드는 프레임워크입니다. 4가지 축으로 구성됩니다.

  • LangChain 라이브러리 — 컴포넌트(프롬프트·모델·파서…)를 체인/에이전트로 결합
  • LangChain 템플릿 — 작업별로 빠르게 배포하는 아키텍처 모음
  • LangServe — 만든 체인을 REST API로 배포
  • LangSmith — 체인 디버깅·테스트·평가·모니터링
📝 오늘의 마음가짐
"정확히 묘사하는 능력"이 새로운 코딩 실력입니다. 오늘은 LangChain 블록 1개(LLM 체인)를 만듭니다.

개발 환경 · API 키

설치 없이 브라우저에서 바로 쓰는 Google Colab으로 실습합니다. 첫 셀에서 라이브러리를 설치하고 버전을 확인합니다.

!pip install langchain>=0.3.0 langchain-openai langchain-community langchain-text-splitters
!pip install faiss-cpu tiktoken   # 벡터검색·토큰계산(이후 사용)

import langchain
from langchain_openai import ChatOpenAI
print(f"LangChain 버전 : {langchain.__version__}")

API 키는 코드에 직접 쓰지 않고 Colab 보안 비밀(🔑 Secrets)에 저장한 뒤 불러옵니다.

import os
from google.colab import userdata
os.environ["OPENAI_API_KEY"] = userdata.get("OPENAI_API_KEY")
⚠️ 키 보안 — 매우 중요
API 키는 절대 공개 코드/깃허브에 올리지 않습니다. 노출 시 요금 폭탄·오남용 위험. 키 = 돈이라는 감각을 처음부터 가집니다.

첫 LLM 호출

모델 객체를 만들고 invoke()로 질문하면 끝입니다. 두 가지 형태가 있습니다.

# (1) 문자열 in → 문자열 out (구형 LLM)
from langchain_community.llms import OpenAI
llm = OpenAI()
print(llm.invoke("안녕하세요"))

# (2) 채팅 모델 (권장) — 응답이 메시지 객체, .content로 텍스트
from langchain_openai import ChatOpenAI
chat = ChatOpenAI(model="gpt-4o-mini")
resp = chat.invoke("지구의 윤년 주기는?")
print(resp.content)
💡 비유
LLM은 "엽서"(문자열→문자열), ChatModel은 "편지봉투+편지지"(역할이 있는 메시지). 앞으로는 주로 ChatModel을 씁니다.

메시지 구조 & temperature

대화는 역할(role)이 있는 메시지로 구성됩니다 — System(개발자가 정한 규칙)·Human(사용자)·AI(응답).

from langchain_core.messages import SystemMessage, HumanMessage

chat = ChatOpenAI(temperature=0)
messages = [
    SystemMessage(content="You are a helpful assistant that translates English to Korean."),
    HumanMessage(content="I love programming"),
]
print(chat.invoke(messages).content)
💡 temperature
0 = 일관·사실적, 1 = 창의·매번 다름. 같은 질문을 0/1로 여러 번 호출해 차이를 직접 확인합니다.

프롬프트 템플릿

반복되는 프롬프트는 변수 {}가 있는 템플릿으로 만들어 값만 바꿔 끼웁니다.

from langchain_core.prompts import ChatPromptTemplate

chat_prompt = ChatPromptTemplate.from_messages([
    ("system", "You are a helpful assistant that translates {input_language} to {output_language}."),
    ("human", "{user_input}"),
])
messages = chat_prompt.format_messages(
    input_language="English", output_language="Korean", user_input="I love programming")
print(ChatOpenAI(temperature=0).invoke(messages).content)

LLM 체인 + 스트리밍

LangChain의 핵심은 체인 — 프롬프트와 모델을 이어 "입력 → 처리 → 출력" 흐름을 만드는 것입니다. (내일 배울 LCEL의 맛보기)

chain = chat_prompt | ChatOpenAI(model="gpt-4o-mini")
print(chain.invoke({"input_language":"English","output_language":"Korean","user_input":"Good morning"}).content)

ChatGPT처럼 실시간 출력 (스트리밍)

from langchain_core.messages import HumanMessage
llm = ChatOpenAI(model="gpt-4o-mini", temperature=0.7)
for chunk in llm.stream([HumanMessage(content="왜 파이썬이 인기있어?")]):
    print(chunk.content, end="", flush=True)

토큰·콜백 모니터링

실무에선 "얼마나 쓰는지"를 추적합니다. 콜백 핸들러로 실행 단계·토큰 사용량·소요 시간을 모니터링할 수 있습니다 (adv_01 노트북의 CallbackHandler 예제).

📝 감각
토큰 = 비용. 입력(prompt)·출력(completion) 토큰이 쌓입니다. 오늘은 "동작 확인" 수준으로만 보고, 모니터링은 콜백으로 자동화할 수 있다는 점만 익힙니다.

오늘의 실습 미션

  1. 나만의 번역기 — 입력 언어·출력 언어를 변수로 받는 ChatPromptTemplate 체인
  2. 나만의 도우미 — system 프롬프트로 역할·말투·금지사항을 정한 비서
  3. 같은 질문을 temperature 0 / 0.7 / 1.0 으로 호출해 결과 비교

과제

  1. Colab에서 첫 LLM 호출 성공 + 스크린샷
  2. 나만의 system 프롬프트 도우미 1개 완성
  3. 스트리밍 출력 동작 확인
