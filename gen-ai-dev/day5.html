5일차 · AI 에이전트 & 자동화 + 배포

AI가 스스로 도구를 쓰게 하는 에이전트와 자동화를 익히고, 만든 결과물을 웹에 배포해 서비스로 공개합니다.

📅 7/3 (금) ⏰ 1~3교시 · 5~9교시 (8H) 📌 종일 · 개념 + 실습

오늘의 시간표

오전 · 1~3교시
개념 + 라이브 데모 — 에이전트·도구 사용·자동화의 개념과 사례
점심
점심 시간
오후 · 5~9교시
실습 + 빌드 — 자동화 워크플로 만들고, 결과물을 웹에 배포해 공개

학습 목표

  • "답만 하는 AI"와 "행동하는 AI(에이전트)"의 차이를 설명할 수 있다.
  • AI가 외부 도구를 호출해 일을 처리하는 흐름을 이해할 수 있다.
  • 반복 작업을 자동화하는 워크플로를 한 개 만들 수 있다.
  • 만든 결과물을 웹에 배포해 URL로 공유할 수 있다.

에이전트란

지금까지는 AI가 말로 답했습니다. 에이전트는 목표를 받으면 스스로 단계를 나눠 행동합니다 — 검색하고, 계산하고, 저장하고.

💡 비유
비서에게 "이번 주 회의 잡아줘"라고 하면 캘린더를 보고, 빈 시간을 찾고, 초대장을 보냅니다. 에이전트가 하는 일이 이와 같습니다.

도구 사용 (Tool Use)

AI에게 "쓸 수 있는 도구 목록"을 주면, 필요할 때 그 도구를 호출합니다.

  • 계산기 — 정확한 수치 계산
  • 검색 — 최신 정보 가져오기
  • 내 함수 — "DB에 저장", "메일 보내기" 같은 우리만의 기능
📝 흐름
AI가 "이 도구를 이렇게 써줘"라고 요청 → 프로그램이 실행 → 결과를 AI에게 돌려줌 → AI가 최종 답. 이 왕복이 핵심입니다.

작업 자동화

매번 손으로 하던 일을 한 번 만들어 두고 반복시킵니다.

  • 이메일/문의를 분류하고 초안 답장 만들기
  • 긴 문서를 요약해 정해진 양식으로 정리
  • 입력이 들어오면 자동으로 처리해 알림 보내기

웹에 배포하기

내 컴퓨터에서만 되는 건 "작품", 누구나 URL로 쓰면 "서비스"입니다.

  1. 코드를 GitHub에 올리기
  2. 무료 호스팅(GitHub Pages 등)에 연결
  3. 받은 주소를 친구에게 공유해 실제로 동작 확인
🔐 배포 시 키 주의
공개 배포에는 API 키가 노출되지 않도록 서버/프록시 또는 비공개 설정을 사용합니다. 강사가 안전한 방법을 안내합니다.

오늘의 과제

  1. 도구 1개를 붙인 에이전트 동작 시연
  2. 내 반복 작업 1개를 자동화 워크플로로 만들기
  3. 결과물을 배포해 URL 확보 → Day 6 발표 준비
