프로젝트 기획

작고 완성된 것이 크고 미완성보다 낫습니다. 한 줄 기획으로 범위를 좁히세요.

📝 기획 한 줄
"이 서비스는 [누구]의 [어떤 문제]를, [내 문서/데이터] 기반 [챗봇/분류기/요약기]로 해결한다."

주제 예시

  • 학과 규정 PDF로 답하는 신입생 안내 RAG 챗봇
  • 동아리 FAQ 문서 기반 Q&A 봇
  • 고객 리뷰 자동 분류 + 요약 도구
  • 내 강의노트로 답하는 나만의 튜터 챗봇

LangChain 구성 선택

기획에 맞춰 필요한 블록만 고릅니다. 배운 것을 레고처럼 조립하세요.

  • 단순 응답/변환 → LLM 체인 (Day 1~2)
  • 분류/태깅 → 분류 체인 + 커스텀 파서 (Day 3)
  • 대화/맥락 → 챗봇 + 대화 기록 (Day 3)
  • 내 문서 기반 → 임베딩 + 벡터스토어 + RAG (Day 4~5)
  • 공개 → Gradio / LangServe (Day 5)

참고 아키텍처 (통합 예제)

가장 인기 있는 형태인 "내 PDF로 답하는 RAG 챗봇 + Gradio 배포" 전체 골격입니다. 주석을 따라 본인 데이터로 바꾸면 됩니다.

# ===== 0) 설치 & 키 =====
# !pip install langchain langchain-openai langchain-community langchain-chroma pypdf gradio
import os
from google.colab import userdata
os.environ["OPENAI_API_KEY"] = userdata.get("OPENAI_API_KEY")

# ===== 1) 문서 적재 → 청크 =====
from langchain_community.document_loaders import PyPDFLoader
from langchain_text_splitters import CharacterTextSplitter

texts, splitter = [], CharacterTextSplitter(separator="\n", chunk_size=1000, chunk_overlap=50)
for f in os.listdir("./pdfs"):
    if f.endswith(".pdf"):
        texts += splitter.split_documents(PyPDFLoader(f"./pdfs/{f}").load())

# ===== 2) 임베딩 → 벡터스토어 → 검색기 =====
from langchain_openai import OpenAIEmbeddings, ChatOpenAI
from langchain_chroma import Chroma
db = Chroma.from_documents(texts, OpenAIEmbeddings())
retriever = db.as_retriever(search_kwargs={"k": 4})

# ===== 3) RAG 체인 (검색→삽입→생성) =====
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser
from langchain_core.runnables import RunnablePassthrough

prompt = ChatPromptTemplate.from_template(
    "아래 [내용]만 근거로 한국어로 답하라. 없으면 모른다고 답하라.\n[내용]\n{context}\n\n[질문]\n{question}")
def format_docs(docs): return "\n\n".join(d.page_content for d in docs)

rag = ({"context": retriever | format_docs, "question": RunnablePassthrough()}
       | prompt | ChatOpenAI(model="gpt-4o-mini", temperature=0.1) | StrOutputParser())

# ===== 4) Gradio UI 배포 =====
import gradio as gr
def respond(message, history): return rag.invoke(message)   # RAG 체인 연결
gr.ChatInterface(respond, title="내 문서 Q&A 봇").launch(share=True)
💡 변형
분류기 프로젝트라면 2~3단계를 "분류 체인"으로, 단순 도우미라면 RAG 없이 "LLM 체인 + Gradio"로 줄이면 됩니다.

구현 체크리스트

  • LCEL 체인이 처음부터 끝까지 동작하는가
  • (RAG라면) 내 문서로 벡터스토어 + retriever를 구성했는가
  • system 프롬프트로 역할·가드레일(모르면 모른다)을 잡았는가
  • Gradio/LangServe로 배포되어 URL로 접속되는가
  • API 키가 노출되지 않았는가 (보안 비밀/환경변수)
  • 핵심 기능 1개가 끝까지 작동하는가 (작아도 완성)

발표 구성 (6분)

  1. 문제 (30초) — 누구의 어떤 불편을 풀었나
  2. 라이브 데모 (2~3분) — 공유 URL로 실제 시연
  3. 구성 설명 (1~2분) — 어떤 LangChain 블록을 어떻게 연결했나
  4. 막힌 점 & 배운 점 (1분) — 실패 공유가 가장 도움됨
⚠️ 흔한 실수
코드만 줄줄 보여주기 / "AI가 다 했어요"라고만 말하기 / 6분 초과. 본인의 의사결정과 데모에 집중하세요.

피어 피드백

동료 발표마다 "좋은 점 1 · 더 해보면 좋을 점 1"을 남깁니다. 잘 듣는 것도 평가 항목입니다.

막판 트러블슈팅

  • 배포 직전 에러 → 전체 노트북을 위에서부터 다시 실행(런타임 재시작 후)
  • share URL 만료 → 발표 직전 다시 launch, 또는 캡처 영상 백업 준비
  • 응답이 느림 → 모델을 gpt-4o-mini로, k를 줄이기
  • 키 노출 걱정 → 화면 공유 전 키 출력 셀 삭제 확인
🤖 발표 백업
데모가 불안하면 정상 동작 장면을 미리 녹화해 두세요. 라이브 실패 시 영상으로 대체.
