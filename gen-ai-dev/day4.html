  • retriever로 첫 RAG 체인을 구성할 수 있다.

임베딩 이해

임베딩은 텍스트를 의미가 담긴 숫자 벡터로 바꿉니다. 의미가 비슷하면 벡터도 가깝습니다.

from openai import OpenAI
client = OpenAI()
resp = client.embeddings.create(model="text-embedding-3-small", input="Hello~ I am Dr.Will")
vec = resp.data[0].embedding
print(len(vec))   # 벡터 차원 수 (모델마다 다름, 예: 1536)

문서 로드 → 분할

긴 문서는 검색·전달이 쉽도록 청크(chunk)로 나눕니다. "1000자짜리 상자"에 담는다고 생각하세요.

import urllib.request
urllib.request.urlretrieve(
  "https://raw.githubusercontent.com/hwchase17/chat-your-data/master/state_of_the_union.txt",
  "state_of_the_union.txt")

from langchain_community.document_loaders import TextLoader
from langchain_text_splitters import CharacterTextSplitter
raw = TextLoader("state_of_the_union.txt").load()
docs = CharacterTextSplitter(chunk_size=1000, chunk_overlap=0).split_documents(raw)
print(len(docs), "개 청크")

벡터스토어 구축

청크를 임베딩해 Chroma에 저장합니다. 청크 1개 → 벡터 1개(예: 1536차원).

from langchain_openai import OpenAIEmbeddings
from langchain_chroma import Chroma
db = Chroma.from_documents(docs, OpenAIEmbeddings())

유사도 검색

질문을 벡터로 바꿔, 가장 가까운 청크들을 찾아옵니다(기본 4개).

query = "What did the president say about Ketanji Brown Jackson"
hits = db.similarity_search(query)
print(hits[0].page_content)
💡 동작 원리
문서→42개 상자→각 상자에 1536차원 벡터. 질문도 벡터로 바꿔(QV) 42개와 유사도(코사인)를 재 가장 가까운 상자 k개를 반환.

retriever & 첫 RAG

검색기를 만들고, 찾은 문서를 프롬프트의 context로 넣으면 RAG(검색→삽입→생성)가 됩니다.

from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser
from langchain_openai import ChatOpenAI

retriever = db.as_retriever(search_kwargs={"k": 3})
prompt = ChatPromptTemplate.from_template(
  "다음 문서를 참고해 답하세요.\n문서:\n{context}\n\n질문:\n{question}")
def format_docs(ds): return "\n\n".join(d.page_content for d in ds)

chain = ({"context": lambda q: format_docs(retriever.invoke(q)), "question": lambda q: q}
         | prompt | ChatOpenAI(model="gpt-4o", temperature=0) | StrOutputParser())
print(chain.invoke("케탄지 브라운 잭슨에 대해 한국어로 답해줘"))

직접 구현·한글·튜닝

  • 직접 구현 — SimpleTextLoader·SimpleVectorStore로 코사인 유사도 내부 동작 체험 (lang_08)
  • 한글 임베딩 — HuggingFace 임베딩으로 한국어 문서 품질 비교
  • 튜닝 — chunk_size·overlap·검색 개수(k)를 바꿔 검색 품질 개선
