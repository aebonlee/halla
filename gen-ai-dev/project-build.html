설계 → 구현 공통 틀

모든 예제는 같은 흐름을 따릅니다. 아이디어를 이 틀에 맞춰 옮기면 됩니다.

  1. 기획 — 누구의 어떤 문제 (한 줄)
  2. 설계 — 어떤 LangChain 블록을 어떤 순서로 (분해)
  3. 구현 — 블록을 코드로 (LCEL 파이프)
  4. 확장 — 더 좋게 만들 다음 한 걸음
💡 공통 준비
모든 예제는 Colab + os.environ["OPENAI_API_KEY"] = userdata.get("OPENAI_API_KEY") 가 되어 있다고 가정합니다.

예제 1 · 문서 Q&A RAG 챗봇

🎯 기획(한 줄)
신입생이 학과 규정 PDF를 일일이 안 읽어도 질문하면 답해주는 챗봇.

설계 — 사용하는 블록

  • 문서 로드: PyPDFLoader
  • 분할: CharacterTextSplitter
  • 임베딩+저장: OpenAIEmbeddings → Chroma
  • 검색: retriever · 생성: RAG 체인

구현

from langchain_community.document_loaders import PyPDFLoader
from langchain_text_splitters import CharacterTextSplitter
from langchain_openai import OpenAIEmbeddings, ChatOpenAI
from langchain_chroma import Chroma
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser
from langchain_core.runnables import RunnablePassthrough

# 1) 문서 → 청크
docs = PyPDFLoader("./pdfs/rules.pdf").load()
chunks = CharacterTextSplitter(chunk_size=1000, chunk_overlap=100).split_documents(docs)

# 2) 임베딩 → 벡터스토어 → 검색기
db = Chroma.from_documents(chunks, OpenAIEmbeddings())
retriever = db.as_retriever(search_kwargs={"k": 4})   # 유사 청크 4개

# 3) RAG 체인 (없으면 모른다고 답 = 환각 방지)
prompt = ChatPromptTemplate.from_template(
    "아래 [내용]만 근거로 한국어로 답하라. 없으면 '문서에 없습니다'라고 답하라.\n"
    "[내용]\n{context}\n\n[질문]\n{question}")
fmt = lambda ds: "\n\n".join(d.page_content for d in ds)

rag = ({"context": retriever | fmt, "question": RunnablePassthrough()}
       | prompt | ChatOpenAI(model="gpt-4o-mini", temperature=0.1) | StrOutputParser())

print(rag.invoke("휴학 신청 기한이 언제야?"))
➕ 확장 아이디어
출처(page) 함께 표시, 여러 PDF 폴더 로드, 대화 기록(이전 질문 맥락) 추가.

예제 2 · 리뷰 감성분석 + 요약

🎯 기획(한 줄)
쇼핑몰 사장이 리뷰 수백 개를 긍/부정으로 자동 분류하고 핵심을 요약해 본다.

설계 — 사용하는 블록

  • 분류: 분류 체인 + 커스텀 파서(0/1)
  • 집계: pandas
  • 요약: LLM 체인 (부정 리뷰만 모아 요약)

구현

import pandas as pd
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import BaseOutputParser, StrOutputParser
from langchain_openai import ChatOpenAI

llm0 = ChatOpenAI(temperature=0)

# (1) 감성 분류 체인
clf_prompt = ChatPromptTemplate.from_messages([
    ("system", "리뷰가 긍정이면 1, 부정이면 0만 출력."),
    ("human", "{t}")])
class IntParser(BaseOutputParser):
    def parse(self, text): return 1 if "1" in text else 0
clf = clf_prompt | llm0 | IntParser()

df = pd.read_csv("reviews.csv")           # review 컬럼
df["sent"] = df["review"].apply(lambda t: clf.invoke({"t": t}))
print(df["sent"].value_counts())          # 긍/부정 개수

# (2) 부정 리뷰만 모아 핵심 요약
neg = "\n".join(df[df.sent == 0]["review"].tolist())
sm_prompt = ChatPromptTemplate.from_template("다음 부정 리뷰들의 공통 불만 3가지를 요약해줘:\n{x}")
summary = (sm_prompt | llm0 | StrOutputParser()).invoke({"x": neg})
print(summary)
➕ 확장 아이디어
별점 예측, 키워드 추출, 막대그래프 시각화(matplotlib), 주간 자동 리포트.

예제 3 · 페르소나 튜터 챗봇

🎯 기획(한 줄)
학생이 모르는 개념을 물어보면, 초등학생도 알아듣게 비유로 설명하는 튜터.

설계 — 사용하는 블록

  • 페르소나: SystemMessage
  • 맥락 유지: MessagesPlaceholder + 세션 기록
  • 표준 관리: RunnableWithMessageHistory

구현

from langchain_core.prompts import ChatPromptTemplate, MessagesPlaceholder
from langchain_core.output_parsers import StrOutputParser
from langchain_core.chat_history import InMemoryChatMessageHistory
from langchain_core.runnables.history import RunnableWithMessageHistory
from langchain_openai import ChatOpenAI

prompt = ChatPromptTemplate.from_messages([
    ("system", "너는 친절한 코딩 튜터다. 어려운 말 대신 일상 비유로 설명하고, 끝에 한 줄 요약을 붙인다."),
    MessagesPlaceholder(variable_name="history"),
    ("human", "{q}")])
chain = prompt | ChatOpenAI(model="gpt-4o-mini", temperature=0.4) | StrOutputParser()

store = {}
def hist(sid):
    store.setdefault(sid, InMemoryChatMessageHistory()); return store[sid]
tutor = RunnableWithMessageHistory(chain, hist,
            input_messages_key="q", history_messages_key="history")

cfg = {"configurable": {"session_id": "stu-1"}}
print(tutor.invoke({"q": "변수가 뭐예요?"}, config=cfg))
print(tutor.invoke({"q": "방금 그거 예시 하나 더"}, config=cfg))   # 앞 맥락 기억
➕ 확장 아이디어
과목별 튜터(수학/영어), 난이도 조절 슬라이더, 학습 로그 저장.

예제 4 · 문의 자동 분류·라우팅

🎯 기획(한 줄)
고객센터로 들어온 문의를 유형별로 자동 분류해 알맞은 안내로 연결.

설계 — 사용하는 블록

  • 분류: 분류 체인 (배송/환불/상품/기타)
  • 라우팅: 분류 결과에 따라 다른 응답 체인 호출
  • 조건 분기: 파이썬 if 또는 RunnableLambda

구현

from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser
from langchain_openai import ChatOpenAI
llm0 = ChatOpenAI(temperature=0)

LABELS = ["배송", "환불", "상품문의", "기타"]
clf = (ChatPromptTemplate.from_messages([
        ("system", f"문의를 {LABELS} 중 하나로만 분류. 라벨 단어만 출력."),
        ("human", "{q}")]) | llm0 | StrOutputParser())

# 유형별 응답 체인 (간단히 프롬프트만 다르게)
def answer_chain(role):
    return (ChatPromptTemplate.from_messages([("system", role), ("human", "{q}")])
            | ChatOpenAI(temperature=0.3) | StrOutputParser())
handlers = {
    "배송": answer_chain("너는 배송 안내 상담원. 배송조회/기간을 안내."),
    "환불": answer_chain("너는 환불 상담원. 환불 절차를 단계로 안내."),
    "상품문의": answer_chain("너는 상품 전문가. 스펙/호환을 안내."),
    "기타": answer_chain("너는 친절한 상담원. 일반 문의에 답한다."),
}

def route(q):
    label = clf.invoke({"q": q}).strip()
    label = label if label in handlers else "기타"
    return label, handlers[label].invoke({"q": q})

print(route("어제 주문했는데 언제 도착해요?"))
➕ 확장 아이디어
신뢰도 낮으면 사람에게 이관, 통계 집계, Day5 RAG와 결합(규정 기반 답변).

예제 5 · 회의록/문서 요약기

🎯 기획(한 줄)
긴 회의록·문서를 붙여넣으면 핵심 요약 + 할 일(액션 아이템)을 뽑아준다.

설계 — 사용하는 블록

  • 구조화 출력: 요약/결정/할일을 정해진 형식으로
  • LLM 체인 + 프롬프트 설계
  • (긴 문서) 청크별 요약 후 합치기(map-reduce 개념)

구현

from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser
from langchain_openai import ChatOpenAI

prompt = ChatPromptTemplate.from_template(
"""다음 회의록을 아래 형식으로 정리해줘.

[3줄 요약]
- ...

[결정 사항]
- ...

[할 일(담당자/기한 추정 포함)]
- ...

회의록:
{text}""")
summarizer = prompt | ChatOpenAI(model="gpt-4o-mini", temperature=0.2) | StrOutputParser()
print(summarizer.invoke({"text": open("meeting.txt").read()}))
➕ 확장 아이디어
JSON 구조화 출력 → 노션/스프레드시트 자동 입력, 긴 문서 map-reduce 요약.

공통 · Gradio 배포

완성한 체인(rag, tutor, route …)에 채팅 UI를 붙여 공유 URL을 만듭니다.

import gradio as gr
# 예: 예제1의 rag 체인을 그대로 연결
def respond(message, history):
    return rag.invoke(message)          # 다른 예제면 해당 체인으로 교체
gr.ChatInterface(respond, title="내 프로젝트 데모").launch(share=True)
🔐 키 보안
공개 배포 시 API 키는 환경변수로만. 화면 공유 전 키 출력 셀 삭제 확인.
