3일차 · 나만의 AI 챗봇 만들기

역할과 맥락을 담은 시스템 프롬프트로 목적이 분명한 대화형 AI 앱을 처음부터 만들어 봅니다.

📅 7/1 (수) ⏰ 1~3교시 · 5~9교시 (8H) 📌 종일 · 개념 + 실습

오늘의 시간표

오전 · 1~3교시
개념 + 라이브 데모 — 목적이 분명한 챗봇 기획, 대화 흐름·페르소나 설계
점심
점심 시간
오후 · 5~9교시
실습 + 빌드 — 노코드/로우코드로 챗봇 만들고 화면에 붙여 동작 확인

학습 목표

  • "누구를 위해, 무엇을 돕는" 챗봇인지 한 문장으로 정의할 수 있다.
  • 대화 이력을 유지해 맥락을 이어가는 챗봇을 만들 수 있다.
  • 입력창·말풍선이 있는 화면에 챗봇을 붙여 동작시킬 수 있다.

챗봇 기획

좋은 챗봇은 기능이 많은 게 아니라 목적이 분명한 챗봇입니다.

📝 한 문장 기획
"이 챗봇은 [누구]를 위해 [어떤 상황]에서 [무엇]을 돕는다."

예시

  • 신입생을 위해 수강신청 시즌에 학사 규정을 안내하는 챗봇
  • 자취생을 위해 냉장고 재료로 요리를 추천하는 챗봇
  • 취준생을 위해 자소서 문장을 다듬어 주는 챗봇

대화 이어가기

한 번 답하고 끝이 아니라 이전 대화를 기억해야 진짜 챗봇입니다. 비결은 어제 배운 메시지 구조 — 이전 user/assistant 메시지를 매번 함께 보냅니다.

messages = [
  { role: "user",      content: "다이어트 식단 추천해줘" },
  { role: "assistant", content: "닭가슴살 샐러드 어때요?" },
  { role: "user",      content: "그거 칼로리는?" }   // "그거"를 이해함
]
💡 길어지면?
대화가 길수록 토큰(비용)이 늘어납니다. 오래된 대화는 요약해 넣는 기법을 소개합니다.

화면 붙이기

오후에는 두 갈래 중 본인에게 맞는 방식으로 만듭니다.

  • 노코드 — 챗봇 빌더 도구에 시스템 프롬프트만 넣어 즉시 완성
  • 로우코드 — 어제 만든 API 호출에 입력창·말풍선 화면을 붙이기 (바이브 코딩 방식으로 Claude에게 요청)
🤝 오후반과 만나는 지점
화면 만들기가 막히면 "HTML로 채팅 화면 만들어줘"처럼 자연어로 요청하세요 — 바이브 코딩의 힘을 그대로 씁니다.

오늘의 과제

  1. 내 챗봇 한 문장 기획서 + 시스템 프롬프트 완성
  2. 맥락을 이어가는 대화 3턴 이상 시연 캡처
  3. 화면에 붙여 친구에게 한 번 써보게 하고 피드백 받기
