5일차 · 미니 프로젝트 ②

개인 소개 웹페이지 발표 — AI에게 어떻게 요청했는가가 핵심입니다.

📅 7/3 (금) ⏰ 5~9교시 (5H) 📌 발표 + 회고

오늘의 시간표

13:00–13:30
최종 점검 + 발표 순서 추첨 배포 URL 작동 확인, 모바일 동작 확인
13:30–15:00
1부 발표 (1~20번) 1인 6분 발표 + 2분 Q&A · 중간 휴식 10분
15:10–16:40
2부 발표 (21~40번) 1인 6분 발표 + 2분 Q&A
16:50–17:30
전체 회고 + 우수상 시상 베스트 페이지 / 베스트 프롬프트 / 베스트 발표 시상
17:30–17:50
다음 단계 안내 + 클로징 심화 트랙 안내, 수료증 전달

오늘의 목표

  • 실제로 작동하는 개인 소개 웹페이지를 인터넷에 공개한다.
  • 구현 과정에서 AI에게 어떻게 요청했는지를 발표의 중심에 둔다.
  • 1주일의 학습을 본인 말로 정리해 다음 단계 방향을 잡는다.

발표 구성 - 6분 안에 들어가야 할 6가지

0:00 - 0:30
① 인사 + URL 공개 "안녕하세요, 제 페이지는 [URL]입니다. QR 코드 띄워서 청중이 바로 접속 가능하게."
0:30 - 1:30
② 페이지 라이브 시연 실제 페이지를 스크롤하며 어떤 섹션이 있는지 보여줌. 다크모드 토글 등 인터랙션 시연.
1:30 - 3:30
③ "AI에게 어떻게 요청했는가" (핵심!) 가장 좋은 결과를 만든 프롬프트 1-2개를 화면에 띄우고 설명.
"처음엔 ___라고 요청 → 결과가 별로 → ___로 다듬으니 잘 됨"
3:30 - 4:30
④ 가장 막혔던 순간 + 해결법 어떤 에러를 만났고, AI와 어떻게 풀었는지. (실패 공유가 청중에게 가장 도움됩니다)
4:30 - 5:30
⑤ 5일 전과 후 "5일 전엔 HTML이 뭔지도 몰랐는데 지금은 ___ 할 수 있다." 본인 변화 한 줄.
5:30 - 6:00
⑥ 마무리 + 청중 질문 "이 페이지에 추가하고 싶은 다음 기능은 ___입니다. 여러분이라면?"
⚠️ 발표 흔한 실수
  • 코드만 보여주기 — 청중은 코드 자체보다 "어떻게 만들었는가"가 궁금합니다.
  • AI가 다 해줬다고 말하기 — 본인이 어떤 의사결정을 했는지가 핵심입니다.
  • 완벽한 척하기 — "막혔던 순간"이 발표의 꽃입니다.
  • 6분 넘기기 — 5분 30초 안에 끝낼 수 있게 미리 한 번 리허설.

평가 루브릭

강사 60% + 동료 평가 40%로 합산합니다.

구현 깊이 25점 JavaScript 인터랙션 1개 이상, 반응형, 다크모드 등 다양한 요소
AI 활용 과정 설명 25점 "어떻게 요청했는지"를 구체적으로 보여주는가, 프롬프트 튜닝 흔적
디자인 완성도 15점 색·여백·폰트의 통일감, 본인 개성이 드러나는가
