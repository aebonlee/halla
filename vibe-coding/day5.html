5일차 · 인터랙션 & 기능 추가

버튼·폼·저장 — 자바스크립트로 페이지에 '움직임'을 더합니다. AI에게 동작을 자연어로 요청해 붙여 봅니다.

📅 7/3 (금) ⏰ 5~9교시 (5H) 📌 실습 중심
이 페이지에서
  • 오늘의 시간표
  • 학습 목표
  • 자바스크립트가 하는 일
  • 버튼·폼 다루기
  • 브라우저에 저장하기
  • 오늘의 과제

오늘의 시간표

5~6교시
개념 + 데모 자바스크립트가 페이지에서 하는 일 / 이벤트의 개념
7~9교시
실습 버튼·폼·저장 기능을 AI에게 요청해 내 페이지에 붙이기

학습 목표

  • HTML/CSS는 "겉모습", 자바스크립트는 "움직임"이라는 역할 차이를 설명할 수 있다.
  • 버튼 클릭·폼 입력 같은 사용자 동작에 반응하는 기능을 붙일 수 있다.
  • localStorage로 입력 내용을 브라우저에 저장해 재방문 시 유지할 수 있다.

자바스크립트가 하는 일

지금까지 만든 페이지는 "보이기만" 했습니다. 자바스크립트(JS)는 여기에 "누르면 반응하는" 움직임을 더합니다.

💡 한 줄 비유
HTML은 뼈대, CSS는 옷, JS는 근육 — 실제로 움직이게 하는 부분입니다.
  • 버튼을 누르면 글자가 바뀐다
  • 폼에 입력하면 화면에 목록이 추가된다
  • 토글을 누르면 다크모드로 바뀐다

버튼·폼 다루기

핵심 개념은 이벤트입니다 — "무슨 일이 일어나면(클릭/입력) → 무엇을 해라".

// "버튼을 누르면 인사말을 보여줘" 를 AI에게 요청하면 이런 코드를 만들어 줍니다
버튼.addEventListener("click", () => {
  화면.textContent = "환영합니다!";
});
📝 바이브 코딩 방식
코드를 외우지 않습니다. "이름을 입력하고 버튼을 누르면 인사하는 기능 만들어줘"처럼 원하는 동작을 자연어로 요청하고, 나온 코드를 붙여 동작을 확인합니다.

브라우저에 저장하기

새로고침해도 입력이 사라지지 않게 하려면 localStorage에 저장합니다. 방명록·메모·할 일 목록 같은 기능의 토대입니다.

⚠️ 알아둘 점
localStorage는 그 브라우저에만 저장됩니다(서버 X). "내 컴퓨터 메모장"이라고 생각하면 됩니다.

오늘의 과제

  1. 내 페이지에 버튼 동작 1개 추가 (예: 좋아요 카운터, 인사 토글)
  2. 입력 폼 + 저장 기능 1개 추가 (예: 방명록 한 줄 남기기)
  3. 막힌 부분의 에러 메시지를 AI에게 그대로 보여주고 해결하기
