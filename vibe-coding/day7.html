7일차 · 배포 & 인터넷 공개

내 컴퓨터에서만 되던 페이지를 인터넷에 올려 URL로 공유합니다. GitHub Pages로 실제 배포.

📅 7/7 (화) ⏰ 5~9교시 (5H) 📌 실습 중심
이 페이지에서
  • 오늘의 시간표
  • 학습 목표
  • 왜 배포하는가
  • GitHub에 올리기
  • GitHub Pages로 공개
  • 오늘의 과제

오늘의 시간표

5~6교시
개념 + 데모 배포란 무엇인가 / GitHub·GitHub Pages 흐름
7~9교시
실습 내 페이지를 GitHub Pages로 배포해 URL 확보 + 모바일 점검

학습 목표

  • "내 컴퓨터에서만 되는 것"과 "인터넷에 공개된 것"의 차이를 안다.
  • GitHub에 코드를 올리고 GitHub Pages로 페이지를 공개할 수 있다.
  • 받은 URL을 휴대폰·친구에게 공유해 실제 동작을 확인할 수 있다.

왜 배포하는가

아무리 잘 만들어도 내 노트북 안에만 있으면 아무도 못 봅니다. URL이 생기는 순간 비로소 "작품"이 "공유 가능한 결과물"이 됩니다.

💡 발표의 핵심
금요일 발표는 "내 URL을 청중이 휴대폰으로 직접 열어보는" 순간이 하이라이트입니다.

GitHub에 올리기

  1. GitHub 계정 만들기 / 로그인
  2. 새 저장소(repository) 생성
  3. 내 파일(HTML/CSS/JS) 업로드 — 드래그&드롭 또는 Cursor에서 연동
📝 막히면
"GitHub에 처음 올리는 절차를 한 단계씩 알려줘"라고 AI에게 물으면 화면에 맞춰 안내해 줍니다.

GitHub Pages로 공개

  1. 저장소 Settings → Pages
  2. 배포 브랜치 선택 → 저장
  3. 1~2분 뒤 생성된 URL 접속 확인
  4. 휴대폰에서도 열어 보고 깨지는 곳 점검
⚠️ 자주 겪는 일
배포 직후엔 잠시 안 보일 수 있습니다(반영 대기). 이미지·파일 경로 대소문자도 흔한 실수 — 안 보이면 경로부터 확인.

오늘의 과제

  1. 내 페이지 GitHub Pages 배포 완료 → URL 확보
  2. 휴대폰에서 접속해 정상 동작 확인
  3. URL을 동료 1명에게 공유하고 피드백 받기
