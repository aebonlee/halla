학습 목표

  • AI가 만든 코드를 한 줄씩 자기 말로 설명할 수 있다.
  • 코드의 일부분만 자연어로 정확히 수정 요청할 수 있다.
  • 에러 메시지를 보고 어디부터 봐야 할지 직관을 갖는다.

"이 코드 설명해줘" - AI를 가르치는 학생처럼 쓰기

AI에게 코드를 설명해달라고 할 때, "무엇을 모르는지"를 같이 알려주면 결과가 훨씬 좋습니다.

실습 ① 한 줄씩 설명 받기

어제 만든 calc.py를 Cursor에 열고, 전체 코드를 선택 (Cmd/Ctrl + A) 후 채팅에 붙여 넣습니다.

이 코드를 분석해줘.

1. 각 줄이 무슨 일을 하는지 한 줄씩 설명 (주석 형태)
2. 사용된 핵심 개념 5가지를 짚어줘 (함수, 조건문, 입력 등)
3. 비전공자가 가장 헷갈릴 만한 부분 3개
4. "이 코드의 한계" — 어떤 입력에서 깨질 수 있는가

실습 ② "마치 10살에게 설명하듯이"

같은 코드를 다음 프롬프트로 또 설명받아 봅니다.

이 코드를 컴퓨터를 처음 배우는 10살에게 설명해줘.
어려운 단어는 비유로 풀어줘.
예시: "변수는 이름표를 붙인 박스 같은 거야."
💡 핵심 학습법
AI에게 코드를 설명받았으면, 그걸 본인이 다시 친구에게 설명할 수 있는지 확인하세요.
입으로 한 번 말해 봐야 진짜 이해한 겁니다.

부분 수정 요청 - 정확한 단어로 말하기

AI는 "이거 좀 고쳐줘"를 들으면 곤란해 합니다. "어디를, 어떻게"를 정확히 말해야 합니다.

실습 ① 어휘 익히기

다음 단어들을 정확히 쓰는 게 핵심입니다.

  • 함수 (function): def 이름():으로 시작하는 한 묶음의 코드
  • 변수 (variable): 값을 저장하는 이름표 (예: name = "Alice")
  • 조건문 (if statement): 만약 ~이면 ~해라
  • 반복문 (loop): 같은 일을 여러 번 (for, while)
  • 인자 (argument): 함수가 받는 입력값
  • 리턴 (return): 함수가 돌려주는 결과

실습 ② 좋은 수정 요청 vs 나쁜 수정 요청

나쁜 요청

"이거 좀 좋게 만들어줘"

좋은 요청

"calculator 함수 안의 if 조건문 4개를 더 깔끔하게 정리해줘.
가능하면 dictionary를 써서 줄여줘.
동작은 같게 유지."

실습 ③ 실전 수정 요청 10가지

본인 코드에 다음을 차례로 시도해 보세요.

  1. "calculator 함수 안에 한국어 주석을 모든 줄에 달아줘."
  2. "변수 a, b 이름을 num1, num2로 바꿔줘 (전체 일관되게)."
  3. "input() 받을 때 잘못된 값이 들어오면 에러 메시지 출력하고 다시 묻게 해줘."
  4. "main() 함수를 만들고 calculator()를 그 안에서 호출하게 해줘."
  5. "print 출력을 모두 한국어 → 영어로 바꿔줘."
  6. "결과 출력을 소수점 둘째 자리까지만 보이게 해줘."
  7. "이 파일을 calc_v2.py로 다시 저장하고, 원본은 건드리지 마."
  8. "코드 맨 위에 docstring으로 이 파일이 뭘 하는지 설명을 붙여줘."
  9. "이 코드를 함수 단위로 잘게 쪼개줘 (입력, 계산, 출력 분리)."
  10. "테스트 함수 3개를 추가해 (덧셈, 나눗셈, 0으로 나누기)."

에러 메시지 해석 - 디버깅의 시작

에러는 적이 아니라 친구입니다. 파이썬은 친절하게도 "어디서, 무엇이 잘못됐는지" 알려줍니다.
다만 영어로 알려주죠.

자주 보는 에러 5가지

① SyntaxError - 문법 오류

SyntaxError: invalid syntax

괄호 안 닫음, 콜론 누락, 따옴표 짝 안 맞음 등. 해당 줄 또는 바로 위 줄을 살피세요.

② NameError - 이름을 모르겠어요

NameError: name 'nubmer' is not defined

오타가 가장 흔한 원인. number를 nubmer로 친 경우.

③ TypeError - 타입이 안 맞아요

TypeError: can only concatenate str (not "int") to str

문자열과 숫자를 더하려고 했을 때. str(숫자)로 변환하거나 f"문자열 {숫자}" 사용.

④ IndentationError - 들여쓰기

IndentationError: expected an indented block

파이썬은 들여쓰기가 문법입니다.
스페이스 4칸이 표준.

⑤ ZeroDivisionError - 0으로 나누기

ZeroDivisionError: division by zero

의도하지 않은 0이 들어왔을 때. if b != 0: 같은 가드가 필요합니다.

황금 디버깅 프롬프트

에러가 났을 때 다음 형식으로 AI에게 물어보세요.

다음 코드를 실행했는데 에러가 났어:

```python
[코드 붙여넣기]
```

에러 메시지는:
```
[에러 메시지 그대로 붙여넣기]
```

내가 시도하려던 일은: [무엇을 하려 했는지]

1. 에러의 진짜 원인이 뭐야?
2. 어디를 어떻게 고쳐야 해?
3. 비슷한 에러를 또 안 만나려면 뭘 조심해야 해?
🔍 디버깅 마인드셋
에러가 났을 때 "내가 못해서"라고 생각하지 마세요. "이 에러를 통해서 새로운 걸 알게 될 거야"가 더 정확합니다.
프로 개발자도 매일 에러를 봅니다.
차이는 빨리 알아보고 빨리 고치는 것뿐.

미니 게임 - 숫자 맞히기

오늘 배운 능력을 활용해 작은 게임을 만들어 봅니다.

요구사항

다음 사양으로 파이썬 숫자 맞히기 게임을 만들어줘:

- 컴퓨터가 1~100 사이 랜덤 숫자 1개 선택
- 사용자에게 입력받아 정답과 비교
- 너무 큼 / 너무 작음 / 정답 안내
- 시도 횟수 카운트
- 정답 시 시도 횟수에 따라 등급 표시
  (1-5번: 천재 / 6-10번: 우수 / 11-20번: 보통 / 21번+: 인내심 좋음)
- 게임 끝나면 다시 할지 묻기
- 한국어 안내, 이모지 활용

발전 과제

기본 게임이 완성되면 다음을 추가해 봅니다 (자연어로 요청).

  • 난이도 선택 (쉬움 1-50, 보통 1-100, 어려움 1-500)
  • 최고 기록 저장 (high_score.txt)
  • 잘못된 입력 처리 (숫자가 아닌 경우)
  • 제한 시간 추가 (10초 안에 답하기)
💡 학습 포인트
여기서 진짜 배워야 할 건 게임 자체가 아닙니다. "내가 원하는 것을 묘사 → AI 코드 → 실행 → 자연어 수정" 사이클을 빠르게 도는 감각입니다.
이 사이클을 30번 돌면, 진짜 본인의 웹사이트도 만들 수 있게 됩니다.

오늘의 과제

  1. 본인이 만든 숫자 맞히기 게임에 기능 2가지 추가하고 화면 캡처 제출
  2. 가장 인상 깊었던 에러 1개와 그것을 어떻게 해결했는지 간단한 메모
  3. 내일은 웹페이지를 만듭니다. 본인이 좋아하는 웹사이트 3개의 스크린샷을 가져 오세요 (디자인 참고용)
