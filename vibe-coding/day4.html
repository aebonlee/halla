4일차 · 웹페이지 만들기 ②

JavaScript로 페이지에 움직임을 넣고, 버튼·입력폼·반응형까지 완성합니다.

📅 7/2 (목) ⏰ 5~9교시 (5H) 📌 JS + 반응형

🎯 학습 목표

  • JavaScript의 "이벤트"라는 개념을 이해한다 (클릭, 입력, 마우스 이동 등).
  • 버튼 / 입력폼 / 다크모드 / 햄버거 메뉴를 자연어 요청으로 추가할 수 있다.
  • 본인의 개인 소개 페이지를 모바일에서도 잘 보이게 마무리한다.

1️⃣ JavaScript는 "반응하는 뇌"

HTML로 뼈대를 만들고 CSS로 옷을 입혔다면, JavaScript는 그 페이지가 반응하게 만듭니다. "사용자가 ○○했을 때, △△을 해라"라는 약속을 코드로 적는 것입니다.

JS 1줄 맛보기

<button onclick="alert('안녕!')">클릭</button>

이 버튼을 누르면 "안녕!" 팝업이 뜹니다. 이게 JS의 가장 단순한 모습입니다.

좀 더 깔끔한 방법 (이벤트 리스너)

<button id="myBtn">클릭</button>

<script>
  const btn = document.getElementById('myBtn');
  btn.addEventListener('click', () => {
    alert('안녕!');
  });
</script>

JS의 3대 동작

  1. 찾기: document.getElementById('id')로 HTML 요소를 찾는다
  2. 듣기: .addEventListener('click', ...)로 이벤트를 기다린다
  3. 바꾸기: .textContent = '새 글자'로 내용을 바꾼다

2️⃣ 실습 ① 버튼 클릭 - 색 토글

Cursor 채팅에 요청하세요.

HTML 한 파일에 다음 동작을 만들어줘:

- 가운데 큰 박스 하나 (200x200, 푸른색)
- 박스 위에 "색 바꾸기" 버튼
- 버튼 클릭할 때마다 박스 색이 [파랑 → 빨강 → 노랑 → 초록 → 파랑]으로 순환
- 박스 안에 현재 색 이름을 한글로 표시

자세한 주석 한국어로.

AI가 만든 코드를 실행해 보고, 다음을 추가 요청해 보세요.

  • "색 바뀔 때 부드럽게 전환되도록 0.3초 transition 추가."
  • "키보드 스페이스바로도 색 바뀌게 해줘."
  • "다음 색 미리보기를 작은 동그라미로 옆에 표시해줘."

3️⃣ 실습 ② 입력폼 - 방명록

방명록 페이지를 만들어줘.

- 이름 입력칸 + 메시지 입력칸 + "남기기" 버튼
- 버튼 누르면 아래 목록에 [이름: 메시지] 형식으로 추가
- 새로 추가되는 항목은 위에 쌓이게
- 빈 칸은 제출 안 되게
- 입력 후 칸은 자동으로 비워지게
- localStorage로 새로고침해도 메시지가 남게

디자인은 깔끔한 카드 스타일, 푸른색 강조.

발전 과제 - 자연어로 다음 기능 추가

  • "각 메시지에 작성 시간(몇 분 전) 표시."
  • "각 메시지 옆에 '❤' 좋아요 버튼. 누를 때마다 카운트 증가."
  • "본인이 남긴 메시지는 삭제 버튼이 보이게."
  • "비속어 필터 - 특정 단어가 들어가면 ***로 가리기."
💡 핵심 학습 포인트
localStorage는 브라우저에 데이터를 저장합니다. 새로고침해도 데이터가 살아 있죠. 이게 백엔드 없이도 작은 앱을 만들 수 있게 해주는 마법입니다.

4️⃣ 실습 ③ 다크모드 토글

요즘 모든 사이트에 있는 다크모드를 본인 페이지에도 추가해 봅니다.

내 자기소개 페이지(about.html)에 다크모드 토글 기능 추가해줘.

요구사항:
- 우측 상단에 🌙 / ☀️ 아이콘 버튼
- 클릭하면 다크 ↔ 라이트 전환
- CSS 변수(--bg, --text 등)를 써서 색을 깔끔하게 관리
- 사용자의 선택을 localStorage에 저장
- 다음에 페이지를 다시 열면 이전 선택 유지
- 전환은 0.3초 부드럽게

잘 안 되면 다음과 같이 디버깅 요청:

다크모드 토글이 작동하지 않아.
버튼을 눌러도 색이 안 바뀌어.
콘솔에는 다음 에러가 떠:
[브라우저 개발자도구 콘솔의 에러 메시지 그대로 붙여넣기]

뭐가 문제이고 어떻게 고쳐야 해?

5️⃣ 실습 ④ 반응형 - 모바일에서도 멋지게

요즘 웹 트래픽의 60%는 모바일에서 옵니다. 반응형이 필수입니다.

핵심 개념 - Media Query

/* 화면 너비가 700px 이하일 때 적용 */
@media (max-width: 700px) {
  .container {
    padding: 16px;
  }
  h1 {
    font-size: 24px;
  }
  .grid {
    grid-template-columns: 1fr;  /* 한 줄에 하나씩 */
  }
}

실전 요청

내 about.html을 다음 조건의 반응형으로 만들어줘:

- 데스크탑(1024px+): 3컬럼 그리드, 좌우 여백 넉넉히
- 태블릿(700-1023px): 2컬럼
- 모바일(700px 이하): 1컬럼, 글자 크기 살짝 줄임
- 모바일에선 상단 메뉴를 햄버거 메뉴(☰)로 변경
- 햄버거 클릭하면 메뉴가 슬라이드로 펼쳐지게

CSS는 기존 스타일과 충돌 없게.

모바일에서 테스트

  1. 본인 페이지를 브라우저에서 열기
  2. 개발자도구 (F12) → 모바일 토글 아이콘 (📱)
  3. iPhone, iPad, Galaxy 등 여러 사이즈로 확인

6️⃣ GitHub Pages로 배포 미리보기

오늘 만든 페이지를 인터넷에 올려 봅시다. 무료로, 5분이면 됩니다.

  1. github.com 가입 (학교 이메일 추천)
  2. "New repository" → 이름 halla-portfolio → Public
  3. 본인 약속 페이지 파일들을 드래그&드롭으로 업로드
  4. Settings → Pages → Source: "Deploy from a branch" → main / root → Save
  5. 1-2분 후 https://[본인아이디].github.io/halla-portfolio로 접속 가능
⚠️ 자주 만나는 문제
  • 파일명은 반드시 index.html이어야 메인으로 인식됩니다 (about.html이면 URL 뒤에 /about.html 붙여야 함)
  • 한글 파일명은 가급적 피하세요 (URL 인코딩 문제)
  • 이미지 경로는 상대 경로 또는 외부 URL 사용
🌐 본인 도메인 만들기 (선택)
이름.github.io라는 이름의 리포지토리를 만들면, 이름.github.io 자체가 도메인이 됩니다. 여기에 본인 페이지를 올리면 명함처럼 깔끔합니다.

📚 오늘의 과제 - 내일 발표 준비

  1. 본인 개인 소개 페이지 최종본 완성하기. 다음이 들어가면 좋습니다:
    • 이름, 학과, 자기소개 한 문장
    • 관심사 / 좋아하는 것
    • 포트폴리오 또는 본인의 작품 (이번 강의에서 만든 것들도 좋음)
    • 연락하기 (이메일, GitHub, SNS)
    • 다크모드 토글 또는 다른 인터랙션 1개 이상
    • 모바일에서 잘 보이게
  2. GitHub Pages로 배포까지 완료. 접속 가능한 URL이 내일 발표 자료입니다.
  3. 발표 준비 — "어떤 프롬프트로 어떤 결과를 얻었는지" 가장 인상 깊은 순간 1개 정하기.
