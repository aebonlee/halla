오늘의 시간표

최종 점검
발표 순서 추첨 + URL 작동 확인 배포 URL·모바일 동작 최종 점검
1부 발표
1~20번 1인 6분 발표 + 2분 Q&A · 중간 휴식
2부 발표
21~40번 1인 6분 발표 + 2분 Q&A
마무리
전체 회고 + 우수상 시상 + 클로징 베스트 페이지/프롬프트/발표 시상, 수료증 전달

오늘의 목표

  • 실제로 작동하는 개인 웹페이지를 인터넷에 공개해 발표한다.
  • 구현 과정에서 AI에게 어떻게 요청했는지를 발표의 중심에 둔다.
  • 2주간의 학습을 본인 말로 정리해 다음 단계 방향을 잡는다.

발표 구성 — 6분 안에 들어가야 할 6가지

0:00–0:30
① 인사 + URL 공개 QR로 청중이 바로 접속하게
0:30–1:30
② 페이지 라이브 시연 섹션·인터랙션(다크모드 등) 시연
1:30–3:30
③ "AI에게 어떻게 요청했는가" (핵심!) 가장 좋았던 프롬프트 1~2개 + 다듬은 과정
3:30–4:30
④ 가장 막혔던 순간 + 해결법 실패 공유가 청중에게 가장 도움됩니다
4:30–5:30
⑤ 9일 전과 후 "9일 전엔 HTML이 뭔지도 몰랐는데 지금은 ___" 본인 변화 한 줄
5:30–6:00
⑥ 마무리 + 청중 질문 "다음에 추가하고 싶은 기능은 ___"
⚠️ 발표 흔한 실수
  • 코드만 보여주기 — 청중은 "어떻게 만들었는가"가 궁금합니다.
  • AI가 다 해줬다고 말하기 — 본인의 의사결정이 핵심입니다.
  • 완벽한 척하기 — "막혔던 순간"이 발표의 꽃입니다.
  • 6분 넘기기 — 5분 30초 안에 끝내도록 미리 리허설.
