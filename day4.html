📋 오늘의 흐름

오전에 발표자료를 30분에 만들고, 오후엔 본인 페이지에 반응하는 동작 + 다크모드 + 반응형을 모두 끝내고 인터넷에 공개합니다.

  • 🌤 09:00-12:00 Gamma 발표자료 + 영상 스크립트 + 5일차 주제 확정
  • 🌙 13:00-18:00 JS 인터랙션 + 다크모드 + 반응형 + 정적 호스팅 배포

🌤 오전 · AI 콘텐츠 제작 (3H)

"AI가 PPT를 자동으로 만들어주는" Gamma를 익히고, 30초 영상 스크립트의 황금 구조를 잡습니다.

시간표

09:00-09:50
1교시 · 실습Gamma로 발표자료 만들기 — 30분 안에 10장
10:00-10:50
2교시 · 실습영상 스크립트 — 유튜브 쇼츠 / 인스타 릴스
11:00-11:50
3교시 · 발표 + 피드백결과물 공유 / 내일 발표 주제 확정

학습 목표

  • Gamma의 AI 기능으로 본인 주제의 발표자료를 처음부터 끝까지 만든다.
  • 유튜브 쇼츠/릴스용 30-60초 영상 스크립트를 후크-본론-CTA 구조로 작성한다.
  • 내일 발표할 미니 프로젝트의 주제와 결과물 형태를 확정한다.

1️⃣ Gamma — "글로 쓰면 슬라이드가 나온다"

첫 발표자료 만들기

  1. gamma.app 로그인
  2. "+ 새로 만들기" → "프레젠테이션" → "AI로 생성하기"
  3. 다음 프롬프트 중 하나로 시도:
"한라대 학생들에게 AI 도구 사용법을 소개하는 발표.
30분 분량, 10장 슬라이드. 친근한 톤. 이모지 활용."

Claude로 더 좋은 입력 만들기

Gamma 결과가 평범하다면, Claude에서 먼저 구조를 잡아 오세요.

(Claude에서)
나는 [주제]에 대해 30분 발표를 해야 해.
다음 구조로 개요를 만들어줘:

1. 후킹 (첫 슬라이드)
2. 문제 제기 (왜 이 이야기를 해야 하는가)
3. 본론 3개 챕터 (메시지 + 근거 + 예시)
4. 실천 가이드 (오늘부터 할 수 있는 일)
5. 마무리 (인상 깊은 한 마디)

각 슬라이드의 제목과 핵심 문장을 줘.

그 결과를 Gamma의 "텍스트로부터 생성하기"에 붙여 넣으면 품질이 크게 올라갑니다.

디자인 다듬기

  • 오른쪽 패널에서 테마 변경
  • 슬라이드별로 이미지 교체 → "AI 생성"
  • 발표자 노트 추가 → "이 슬라이드에서 강조할 한 마디"
  • 내보내기 → PDF / PPT / 웹 링크
⏱️ 실측 비교
예전엔 10장 슬라이드 = 2-3시간. Gamma + Claude 조합으로 20-30분에 같은 품질.

2️⃣ 영상 스크립트 — 30초가 한 편

황금 구조 - 후크 / 본론 / CTA

  1. 후크 (3초): 손가락이 멈추게 하는 첫 마디
  2. 본론 (15-25초): 정보 1개 또는 반전 1개
  3. CTA (2-5초): "팔로우 / 댓글 / 좋아요" 유도

본인 전공 소개 30초 스크립트

유튜브 쇼츠용 30초 영상 스크립트를 만들어줘.

주제: 한라대 [본인 전공] 신입생에게
       1년 차이 선배가 하는 솔직한 조언
청중: 18-20세 새내기
톤: 친근하고 약간 장난스러움
형식:
- (0-3초) 후크: 충격/호기심 자극 한 마디
- (3-25초) 본론: 진짜 도움 되는 팁 2가지
- (25-30초) CTA: 댓글 유도

각 구간 [화면 / 자막 / 음성]을 표로.

같은 주제, 다른 길이

  • 30초 쇼츠
  • 3분 유튜브 영상 (정보 밀도 ↑)
  • 10분 강의 영상 (개론 + 챕터 3개 + 정리)

3️⃣ AI 콘텐츠 제작 원칙

✓ 출처와 사실 확인

인용·통계·인명은 반드시 출처 확인. Claude는 친절히 "추정"이라고 알려주는 편이지만 100% 신뢰는 금물.

✓ 저작권과 라이선스

  • Canva AI·Gamma·Claude 결과물은 일반적으로 상업 사용 가능 (각 약관 확인 필수)
  • 실명·실제 브랜드명 이미지는 가급적 피하기

✓ AI 사용 표기

AI 사용 비중이 높다면 어떻게 사용했는지 솔직히 밝히기. "AI를 썼다"가 아니라 "어떻게 잘 썼는가"가 평가받는 시대.

4️⃣ 5일차 발표 주제 확정

금요일 미니 프로젝트 ①은 "내가 만든 AI 활용 콘텐츠"입니다. 오늘 3교시에 본인 주제 확정.

주제 예시 (자유 선택)

  • 📚 "AI로 시험기간 1주일 살아남기"
  • 🍕 "AI 비서로 자취 1달 살아보기"
  • 🎨 "AI로 인스타 계정 운영 1주일"
  • 💼 "AI로 자기소개서 5번 다듬기"
  • 🏔️ "AI가 추천한 강원도 1박 2일 여행"
  • 📝 "내 전공에 AI가 들어왔을 때"

결과물 형태 (택 1)

  • 📊 Gamma 발표 슬라이드 8-10장
  • 📄 Claude 워크플로 문서 (PDF · 3페이지)
  • 🎬 영상 스크립트 + 캡처 모음 (1분)
  • 🎨 인포그래픽 1장 (Canva AI)

📚 오전 과제

  1. 발표 주제 1개 확정 + 결과물 초안 90% 완성
  2. 발표 시간 5분 안에 끝낼 수 있도록 시간 체크
  3. 청중에게 던질 질문 1개 준비

🌙 오후 · 웹페이지 만들기 ② (5H)

JavaScript로 페이지가 반응하게 만들고, 다크모드·반응형까지 완성한 다음 인터넷에 공개합니다.

학습 목표

  • JavaScript의 "이벤트"라는 개념을 이해한다.
  • 버튼·입력폼·다크모드·햄버거 메뉴를 자연어 요청으로 추가할 수 있다.
  • 본인의 페이지를 모바일에서도 잘 보이게 마무리하고 인터넷에 공개한다.

1️⃣ JavaScript는 "반응하는 뇌"

JS 1줄 맛보기

<button onclick="alert('안녕!')">클릭</button>

좀 더 깔끔한 방법 (이벤트 리스너)

<button id="myBtn">클릭</button>

<script>
  const btn = document.getElementById('myBtn');
  btn.addEventListener('click', () => {
    alert('안녕!');
  });
</script>

JS의 3대 동작

  1. 찾기: document.getElementById('id')
  2. 듣기: .addEventListener('click', ...)
  3. 바꾸기: .textContent = '새 글자'

2️⃣ 실습 ① 버튼 클릭 - 색 토글

HTML 한 파일에 다음 동작을 만들어줘:

- 가운데 큰 박스 (200x200, 푸른색)
- 박스 위에 "색 바꾸기" 버튼
- 클릭할 때마다 박스 색이 [파랑 → 빨강 → 노랑 → 초록 → 파랑] 순환
- 박스 안에 현재 색 이름을 한글로 표시
- 자세한 한국어 주석.

발전 요청

  • "색 바뀔 때 0.3초 transition 추가."
  • "키보드 스페이스바로도 색 바뀌게 해줘."
  • "다음 색 미리보기를 작은 동그라미로 옆에 표시."

3️⃣ 실습 ② 입력폼 - 방명록

방명록 페이지를 만들어줘.

- 이름 입력칸 + 메시지 입력칸 + "남기기" 버튼
- 버튼 누르면 아래 목록에 [이름: 메시지] 추가
- 새 항목은 위에 쌓이게
- 빈 칸은 제출 안 되게
- 입력 후 칸은 자동으로 비워짐
- localStorage로 새로고침해도 메시지가 남게

디자인: 깔끔한 카드, 푸른색 강조.

발전 과제

  • "각 메시지에 작성 시간(몇 분 전) 표시."
  • "❤ 좋아요 버튼. 누를 때마다 카운트 증가."
  • "본인이 남긴 메시지는 삭제 버튼이 보이게."
💡 localStorage 마법
브라우저에 데이터를 저장. 새로고침해도 살아 있죠. 백엔드 없이도 작은 앱을 만들 수 있게 해줍니다.

4️⃣ 실습 ③ 다크모드 토글

내 about.html에 다크모드 토글 추가해줘.

- 우측 상단에 🌙 / ☀️ 아이콘 버튼
- 클릭하면 다크 ↔ 라이트 전환
- CSS 변수(--bg, --text)로 색 관리
- localStorage에 선택 저장
- 다음에 페이지 열면 이전 선택 유지
- 전환은 0.3초 부드럽게

안 되면 디버깅 요청

다크모드 토글이 작동 안 해.
버튼 눌러도 색이 안 바뀌어.
콘솔 에러:
[브라우저 개발자도구 콘솔의 에러]

뭐가 문제이고 어떻게 고쳐야 해?

5️⃣ 반응형 - 모바일에서도 멋지게

요즘 웹 트래픽의 60%는 모바일. 반응형이 필수입니다.

Media Query

/* 화면 너비 700px 이하일 때 */
@media (max-width: 700px) {
  .container { padding: 16px; }
  h1 { font-size: 24px; }
  .grid { grid-template-columns: 1fr; }
}

실전 요청

내 about.html을 다음 반응형으로:

- 데스크탑(1024px+): 3컬럼 그리드, 좌우 여백 넉넉히
- 태블릿(700-1023px): 2컬럼
- 모바일(700px 이하): 1컬럼, 글자 살짝 작게
- 모바일에선 상단 메뉴 → 햄버거 메뉴(☰)
- 햄버거 클릭 시 메뉴 슬라이드 펼침

모바일 테스트

  1. 브라우저에서 페이지 열기
  2. 개발자도구 (F12) → 모바일 토글 (📱)
  3. iPhone, iPad, Galaxy 여러 사이즈로 확인

6️⃣ 정적 호스팅으로 배포

오늘 만든 페이지를 인터넷에 올려 봅시다. 무료로, 5분이면 됩니다.

방법 ① Netlify Drop (가장 쉬움)

  1. app.netlify.com/drop 접속
  2. 본인 페이지 폴더를 드래그&드롭
  3. 약 30초 후 본인 URL 발급 (예: amazing-name-123.netlify.app)

방법 ② Vercel

  1. vercel.com 가입
  2. "New Project" → 폴더 업로드 또는 Git 연결
  3. 자동 빌드 후 URL 제공
⚠️ 자주 만나는 문제
  • 파일명은 반드시 index.html이어야 메인으로 인식됩니다.
  • 한글 파일명은 피하세요 (URL 인코딩 문제)
  • 이미지 경로는 상대 경로 또는 외부 URL

📚 오후 과제 - 내일 발표 준비

  1. 본인 페이지 최종본 완성. 다음 요소 포함:
    • 이름, 학과, 자기소개
    • 관심사 / 좋아하는 것
    • 포트폴리오 또는 본인 작품
    • 연락하기
    • 다크모드 또는 인터랙션 1개 이상
    • 모바일에서 잘 보임
  2. 정적 호스팅 배포 완료. 접속 가능한 URL이 내일 발표 자료.
  3. "어떤 프롬프트로 어떤 결과를 얻었는지" 가장 인상 깊은 순간 1개 정하기.
