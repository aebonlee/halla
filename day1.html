📋 오늘의 흐름

1️⃣ AI는 도대체 무엇인가

AI(Artificial Intelligence)는 한 마디로 "사람의 지능을 흉내 내는 기계"입니다. 영화 속 자아 있는 로봇이 아니라, "특정 일을 사람만큼 잘하는 프로그램"이 더 정확합니다.

이미 일상에 들어와 있는 AI

  • 📱 스마트폰 얼굴 인식 — 잠금 해제할 때마다 AI가 일하고 있습니다.
  • 🛒 쿠팡 추천 상품 — "이 상품을 본 사람은 이것도 봤어요"의 정체.
  • 🎵 스포티파이 / 유튜브 추천 — 내 취향을 학습한 AI.
  • 🚗 네이버 지도 도착 시간 예측 — 교통량 데이터를 학습한 AI.
  • 💬 ChatGPT, Claude — 대화하는 AI (생성형 AI).

2️⃣ AI · 머신러닝 · 딥러닝 · 생성형 AI

이 네 단어는 동의어가 아닙니다. 큰 원 안에 작은 원이 들어 있는 관계입니다.

한 줄씩 정리

  • AI — 분야 이름. "지능을 흉내 내는 모든 기술" (1956년~)
  • 머신러닝 (ML) — 방법론. "데이터 보여주고 기계가 스스로 패턴 찾기"
  • 딥러닝 — ML의 한 갈래. 인공 신경망을 깊게 쌓아 학습
  • 생성형 AI — 응용 분야. 글·그림·영상·음악을 만들어내는 AI
📝 외울 만한 한 줄
"AI는 분야 이름, ML은 방법론, 딥러닝은 강력한 도구, 생성형 AI는 그 도구로 만든 응용 분야"

3️⃣ ChatGPT vs Claude — 직접 비교 실습

둘 다 생성형 AI지만, 만든 회사도 성격도 다릅니다. 직접 비교해 봅시다.

가입

  1. ChatGPT: chat.openai.com → 구글/네이버 계정으로 가입
  2. Claude: claude.ai → 구글/이메일로 가입
⚠️ 안내
본 강의에서는 Claude 3개월 구독이 필수입니다. 학생 단체 할인 안내는 강사가 별도 공지합니다.

같은 질문 던지기

두 AI에게 다음 4가지를 똑같이 물어보고 비교해 보세요.

  1. "한라대학교에 대해 알고 있는 모든 것을 알려줘."
  2. "내일 한라산 등반 계획을 짜줘. 초보자 기준."
  3. "피보나치 수열을 파이썬으로 구현해줘. 주석 포함."
  4. "강원도 사투리로 자기소개를 만들어줘."

비교 포인트

  • 어느 쪽이 더 정확한가?
  • 어느 쪽이 더 친절하게 답하는가?
  • 어느 쪽이 한국어를 더 자연스럽게 쓰는가?
  • 거짓 정보(환각)는 어디서 더 자주 나오는가?
💡 강사 코멘트
Claude는 긴 문서와 코드에 강하고, ChatGPT는 이미지·웹검색 같은 부가 기능이 풍부합니다. "어떤 일에 어떤 도구가 잘 맞는다"는 감각이 중요합니다.

4️⃣ AI가 바꾸는 직업 세계

📌 핵심 명제
"AI가 사람을 대체하는 게 아니라, AI를 잘 쓰는 사람이 그렇지 않은 사람을 대체합니다."

이미 바뀌고 있는 분야

  • 마케팅 / 콘텐츠 — 카피라이팅, 이미지 생성, 영상 스크립트
  • 고객 응대 / CS — 1차 문의 자동화, 챗봇
  • 번역 / 통역 — 실시간 번역 품질이 사람급
  • 개발 — 코드 자동 완성, 자연어로 코딩 (바이브 코딩)
  • 법률 / 회계 — 판례 검색, 문서 요약, 초안 작성
  • 의료 — 영상 판독, 진단 보조, 진료 기록 자동화

토론 활동 (30분)

  1. 본인 전공/관심 분야에서 AI가 가장 먼저 대체할 일은?
  2. 반대로, AI가 절대 대체할 수 없을 일은? 왜?
  3. 5년 뒤 본인 직업 환경은 어떻게 바뀌어 있을까요?

  2. "내가 자주 하는 일 3가지" 메모해 오기 (내일 사용)

1️⃣ 바이브 코딩이란

2025년 2월, OpenAI 공동창업자였던 Andrej Karpathy가 트위터에 이렇게 썼습니다.

📜 원문
"There's a new kind of coding I call 'vibe coding', where you fully give in to the vibes, embrace exponentials, and forget that the code even exists."

번역: "내가 '바이브 코딩'이라고 부르는 새로운 코딩이 있다. 흐름에 몸을 맡기고, AI의 폭발적 능력을 받아들이고, 코드가 존재한다는 사실조차 잊는 코딩이다."

전통 코딩 vs 바이브 코딩

전통 코딩
  • 언어 문법을 외운다
  • 한 줄 한 줄 직접 친다
  • 구글·스택오버플로 검색
  • 에러 메시지를 직접 해석
바이브 코딩
  • 원하는 결과를 묘사한다
  • AI가 코드를 만든다
  • AI에게 직접 묻는다
  • 에러를 AI에게 보여주고 고친다

그럼 코딩을 안 배워도 되나요?

배워야 합니다. 하지만 "외우는 게 아니라 읽는 능력"이 중요해졌습니다. AI가 만든 코드가 무슨 일을 하는지 알아야 수정도 하고, 잘못된 부분도 찾을 수 있기 때문입니다.

💡 새 코딩 실력의 정의
  1. 원하는 것을 정확히 묘사하는 능력 (= 좋은 프롬프트)
  2. AI가 만든 코드를 읽고 이해하는 능력 (= 독해)
  3. 안 되는 것을 알아채고 다시 요청하는 능력 (= 디버깅 마인드)

2️⃣ 환경 세팅 - 30분 안에 끝내기

① Claude 가입 + Pro 구독 확인

  1. claude.ai 접속 → 구글 또는 이메일로 가입
  2. 오른쪽 상단 프로필 → "Subscription" 클릭
  3. "Claude Pro" 또는 "Team" 플랜 활성화 확인

② Cursor 에디터 설치

  1. cursor.sh 접속 → "Download" → 본인 OS
  2. 설치 후 첫 실행 → "Sign in" 본인 이메일로 가입 (무료 플랜으로 충분)
  3. 설정 (⚙️) → "Models" → "claude-opus" 또는 "claude-sonnet" 선택

③ 작업 폴더 + 첫 파이썬 파일

  1. 바탕화면에 halla-vibe 폴더 생성
  2. Cursor에서 "File" → "Open Folder" → 그 폴더 선택
  3. 새 파일 hello.py 만들고 다음 입력:
print("안녕, 한라대학교!")
print("나는 오늘부터 바이브 코더야.")
  1. 저장 (Ctrl/Cmd + S)
  2. 오른쪽 위 "▶️ Run" 또는 터미널에서 python hello.py
  3. 결과가 출력되면 환경 세팅 완료

3️⃣ 첫 바이브 코딩 - 계산기 만들기

Cursor의 AI 채팅 (Ctrl/Cmd + L)을 엽니다.

자연어로 요청

파이썬으로 간단한 계산기를 만들어줘.
- 사용자에게 두 숫자를 입력받기
- 사칙연산 중 어떤 걸 할지 선택받기 (+, -, *, /)
- 결과를 보기 좋게 출력
- 0으로 나누는 경우는 에러 처리
- 한국어 안내 문구

이어서 자연어 수정 요청

  • "한 번 계산하고 끝내지 말고, 사용자가 '종료'라고 입력할 때까지 반복하게 해줘."
  • "계산 결과를 history.txt 파일에 기록해줘."
  • "%(나머지)와 **(거듭제곱) 연산도 추가해줘."
🔁 바이브 코딩의 핵심 사이클
자연어 요청 → AI 코드 생성 → 실행 → 결과 보기 → 자연어로 수정 → 반복
이 사이클이 빠르게 돌수록 좋은 바이브 코더입니다.

4️⃣ Cursor 핵심 단축키 5가지

단축키 (Mac / Win) 기능
Cmd/Ctrl + LAI 채팅 열기
Cmd/Ctrl + K선택한 부분 인라인 생성/수정
Cmd/Ctrl + IComposer (여러 파일 동시 수정)
TabAI 자동 완성 수락
Cmd/Ctrl + \화면 분할 (코드 + 채팅 동시 보기)

📚 오후 과제

  1. 오늘 만든 계산기에 본인이 직접 기능 1가지 더 추가하기 (예: 평균, 절댓값)
  2. 내일 사용할 Python 파일 1개 미리 만들어 오기 - "주사위 게임" 또는 "가위바위보". Claude에게 자연어로 요청.
