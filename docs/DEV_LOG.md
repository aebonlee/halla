### 설계 결정

1. **정적 HTML 선택**: React/Vite도 고려했으나, 학생들이 코드를 직접 볼 수 있고 GitHub Pages 배포가 가장 단순한 정적 사이트로 결정. 강의 내용 자체가 "HTML/CSS/JS 기초"이므로 자체 코드가 학습 자료로도 기능.
2. **공통 헤더/푸터**: 컴포넌트 시스템 없이 각 페이지에 직접 작성. 페이지 수가 14개로 적고, 학생이 코드를 들춰볼 때도 단순함이 유리.
3. **Pretendard 폰트**: 한글 가독성 최우선. CDN으로 로드.
4. **색상 토큰**: AI 트랙은 청록(#06b6d4), 바이브 트랙은 분홍(#ec4899)으로 시각 구분.
5. **모바일 우선**: 학생 절반 이상이 모바일로 강의 자료 열어볼 것을 가정하고 반응형 적극 적용.

### 콘텐츠 작성 원칙

- **5일 × 2트랙 = 10개 세션마다 일관된 구조** 적용
  - 시간표 → 학습 목표 → 본문 챕터 → 과제 → 이전/다음 네비
- **시간표는 50분 단위로 분할** (한국 대학 표준 1교시)
- **콘텐츠는 "왜 이걸 배우는가"부터 시작**하고 항상 실습으로 마무리
- **각 세션 끝에 구체적 과제**를 적어 다음 차시 연결성 확보
- **코드 예시는 비전공자도 따라할 수 있게 한국어 주석 권장**
