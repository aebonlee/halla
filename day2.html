📋 오늘의 흐름

오전엔 같은 일을 시키더라도 5배 다른 결과를 받는 법을 익히고, 오후엔 AI가 만든 코드를 자기 말로 설명할 수 있게 만듭니다.

  • 🌤 09:00-12:00 좋은 프롬프트의 3대 무기 + 5가지 템플릿
  • 🌙 13:00-18:00 코드 설명 받기 + 수정 + 에러 해석 + 미니 게임

🌤 오전 · 프롬프트 기초 (3H)

"AI는 마음을 읽지 못합니다." 좋은 프롬프트는 본인이 누구이고, 왜 이게 필요하며, 어떤 결과를 원하는지를 모두 담은 한 문단입니다.

시간표

09:00-09:50
1교시 · 강의 + 시연좋은 프롬프트 vs 나쁜 프롬프트
10:00-10:50
2교시 · 실습역할·맥락·출력 형식 3대 무기 익히기
11:00-11:50
3교시 · 미니 챌린지"내가 자주 하는 일"을 프롬프트로 만들기

학습 목표

  • 프롬프트 차이가 결과에 얼마나 영향을 주는지 직접 체감한다.
  • 역할·맥락·출력 형식 세 요소를 의식적으로 넣어 프롬프트를 작성할 수 있다.
  • 본인의 일상 업무 한 가지를 AI 프롬프트로 자동화할 수 있다.

1️⃣ 좋은 프롬프트 vs 나쁜 프롬프트

예시 ① 자기소개

❌ 나쁜

"자기소개 써줘"

✅ 좋은

"너는 IT 회사 신입 채용 담당자야. 한라대 컴공과 24세 지원자가 IT 스타트업에 보낼 자기소개서를 1000자 이내, 진취적이지만 겸손한 톤으로. 강점 3가지를 항목으로 정리해줘."

예시 ② 학습 도움

❌ 나쁜

"미분 설명해줘"

✅ 좋은

"고등 수학을 거의 까먹은 비전공 대학 1학년에게 미분이 뭔지 일상생활 예시 2가지로 설명해줘. 수식 최소화, 마지막에 30초 안에 외울 한 줄 요약."

💡 핵심 통찰
AI는 마음을 못 읽습니다. "내가 누구인지, 왜 이게 필요한지, 어떤 결과를 원하는지"를 명확히 알려줄수록 결과가 폭발적으로 좋아집니다.

2️⃣ 프롬프트의 3대 무기

① 역할 부여 (Role)

AI에게 어떤 사람인 것처럼 답하라고 지시합니다.

"너는 10년차 마케팅 디렉터야."
"너는 친절한 초등학교 선생님이야."
"너는 까다로운 면접관이야."

② 맥락 제공 (Context)

지금 상황, 대상 독자, 목적을 알려줍니다.

"나는 비전공자야."
"청중은 50대 이상 어르신들이야."
"이메일은 거래처 부장님께 보낼 거야."

③ 출력 형식 지정 (Format)

결과물의 길이·구조·스타일을 미리 정합니다.

"500자 이내로 답해줘."
"표 형식으로 정리해줘."
"불릿 5개로 요약해줘."
"JSON 형식으로 줘."
🧪 황금 공식
[역할] + [상황·맥락] + [구체적 요청] + [출력 형식] = 좋은 프롬프트

3️⃣ 자주 쓰는 프롬프트 템플릿 5가지

① 요약 템플릿

다음 글을 [3줄 / 5줄 / 한 문단]으로 요약해줘.
대상 독자는 [누구]야.
[딱딱한 / 친근한] 톤으로.

---
(요약할 글)

② 이메일 작성 템플릿

나는 [내 신분/이름]이야.
[받는 사람]에게 [목적]에 대한 이메일을 쓰려고 해.
배경 상황은 [상황]이야.
[격식 / 친근하게] [몇 자] 정도로 작성해줘.

③ 비교 분석 템플릿

[A]와 [B]를 다음 5가지 기준으로 비교해줘.
1. [기준1]  2. [기준2]  ...
표로 정리한 다음, 상황별 추천을 알려줘.

④ 학습 도우미 템플릿

나는 [내 수준]이야.
[배우고 싶은 개념]을 이렇게 설명해줘:
1. 일상 비유 1개
2. 가장 간단한 예시 1개
3. 흔한 오해 1개
4. 한 줄 요약

⑤ 브레인스토밍 템플릿

[상황]에 대한 아이디어 10개가 필요해.
조건: [예산/시간/대상]
평범한 5개 + 좀 엉뚱한 5개를 섞어서 줘.
각각 1줄 설명.

📚 오전 과제

  1. 오늘 만든 프롬프트 템플릿 1개로 실제 본인 업무·과제에 적용해 before/after 비교 문서 작성
  2. Canva AI 계정 가입 (canva.com) — 내일 사용

🌙 오후 · 자연어로 코드 읽기 (5H)

AI가 만든 코드를 줄 단위로 이해하고, 자연어로 수정·디버깅하는 능력을 익힙니다.

시간표

13:00-13:50
5교시 · 실습어제 만든 Python 파일을 Claude에게 한 줄씩 설명 요청
14:00-14:50
6교시 · 실습코드 부분 수정 요청 (이름 바꾸기, 주석 달기)
15:00-15:50
7교시 · 실습일부러 코드 망가뜨리고 에러 메시지 해석
16:00-16:50
8교시 · 실습미니 게임 "숫자 맞히기" 만들기
17:00-17:50
9교시 · 정리오늘 본 에러 패턴 5가지 정리

학습 목표

  • AI가 만든 코드를 한 줄씩 자기 말로 설명할 수 있다.
  • 코드의 일부분만 자연어로 정확히 수정 요청할 수 있다.
  • 에러 메시지를 보고 어디부터 봐야 할지 직관을 갖는다.

1️⃣ "이 코드 설명해줘"

어제 만든 calc.py를 Cursor에 열고, 전체 코드를 선택 후 채팅에 붙입니다.

이 코드를 분석해줘.

1. 각 줄이 무슨 일을 하는지 한 줄씩 설명 (주석 형태)
2. 사용된 핵심 개념 5가지 짚어줘
3. 비전공자가 가장 헷갈릴 부분 3개
4. "이 코드의 한계" — 어떤 입력에서 깨질 수 있는가

"10살에게 설명하듯이"

이 코드를 컴퓨터 처음 배우는 10살에게 설명해줘.
어려운 단어는 비유로 풀어줘.
예시: "변수는 이름표를 붙인 박스 같은 거야."
💡 핵심 학습법
AI에게 설명받았으면, 그걸 본인이 다시 친구에게 설명할 수 있는지 확인하세요. 입으로 말해 봐야 진짜 이해한 겁니다.

2️⃣ 부분 수정 요청

AI는 "이거 좀 고쳐줘"를 들으면 곤란합니다. "어디를, 어떻게"를 정확히 말해야 합니다.

꼭 알아야 할 어휘

  • 함수 (function): def 이름():으로 시작하는 한 묶음
  • 변수 (variable): 값을 저장하는 이름표
  • 조건문 (if): 만약 ~이면 ~해라
  • 반복문 (loop): 같은 일을 여러 번 (for, while)
  • 인자 / 리턴: 함수가 받는 입력 / 돌려주는 결과

실전 수정 요청 10가지

  1. "calculator 함수에 한국어 주석을 모든 줄에 달아줘."
  2. "변수 a, b 이름을 num1, num2로 바꿔줘 (전체 일관되게)."
  3. "잘못된 값이 들어오면 에러 메시지 출력하고 다시 묻게 해줘."
  4. "main() 함수를 만들고 calculator()를 그 안에서 호출하게."
  5. "print 출력을 모두 한국어 → 영어로."
  6. "결과를 소수점 둘째 자리까지만."
  7. "이 파일을 calc_v2.py로 저장하고 원본 건드리지 마."
  8. "맨 위에 docstring으로 이 파일이 뭘 하는지 설명."
  9. "코드를 함수 단위로 잘게 쪼개줘 (입력·계산·출력 분리)."
  10. "테스트 함수 3개 추가 (덧셈, 나눗셈, 0으로 나누기)."

3️⃣ 에러 메시지 해석

에러는 적이 아니라 친구입니다. 파이썬은 친절하게 어디서, 무엇이 잘못됐는지 알려줍니다.

자주 보는 에러 5가지

① SyntaxError

SyntaxError: invalid syntax

괄호 안 닫음, 콜론 누락, 따옴표 짝 안 맞음. 해당 줄 또는 바로 위 줄을 살피세요.

② NameError

NameError: name 'nubmer' is not defined

오타가 가장 흔한 원인.

③ TypeError

TypeError: can only concatenate str (not "int") to str

문자열과 숫자를 더하려고 했을 때. str(숫자) 변환 또는 f-string 사용.

④ IndentationError

IndentationError: expected an indented block

파이썬은 들여쓰기가 문법. 스페이스 4칸 표준.

⑤ ZeroDivisionError

ZeroDivisionError: division by zero

if b != 0: 같은 가드가 필요합니다.

황금 디버깅 프롬프트

다음 코드를 실행했는데 에러가 났어:

```python
[코드 붙여넣기]
```

에러: [메시지 그대로]
내가 시도하려던 일: [무엇을]

1. 에러의 진짜 원인은?
2. 어디를 어떻게 고쳐야 해?
3. 비슷한 에러를 또 안 만나려면?

4️⃣ 미니 게임 - 숫자 맞히기

다음 사양의 파이썬 숫자 맞히기 게임을 만들어줘:

- 컴퓨터가 1~100 사이 랜덤 숫자 선택
- 사용자 입력 → 정답과 비교
- 너무 큼 / 작음 / 정답 안내
- 시도 횟수 카운트
- 등급 표시: 1-5번 천재 / 6-10 우수 / 11-20 보통 / 21+ 인내심
- 게임 끝나면 다시 할지 묻기
- 한국어 안내, 이모지 활용

발전 과제 (자연어로 추가)

  • 난이도 선택 (쉬움 1-50, 보통 1-100, 어려움 1-500)
  • 최고 기록 저장 (high_score.txt)
  • 잘못된 입력 처리 (숫자가 아닌 경우)
  • 제한 시간 추가 (10초 안에 답하기)

📚 오후 과제

  1. 본인 숫자 맞히기 게임에 기능 2가지 추가 + 화면 캡처 제출
  2. 인상 깊었던 에러 1개와 해결법 메모
  3. 본인이 좋아하는 웹사이트 3개 스크린샷 (내일 디자인 참고용)
