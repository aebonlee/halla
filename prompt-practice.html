프롬프트란 무엇인가

AI에게 원하는 결과를 얻기 위해 전달하는 입력 텍스트를 프롬프트(Prompt)라고 합니다. 단순한 질문이 아니라 AI와의 소통 인터페이스입니다.

한 줄 정리
좋은 프롬프트의 핵심은 구체성 · 명확성 · 맥락 제공 세 가지입니다.

프롬프트 4대 구성요소

좋은 프롬프트는 다음 네 가지를 모두 담고 있습니다.

구성요소 설명 예시
역할 (Role) AI에게 부여할 전문가 역할 당신은 10년차 마케팅 디렉터입니다
맥락 (Context) 배경 정보 · 상황 · 대상 20대 여성 대상 신규 화장품 출시
지시 (Instruction) 구체적인 작업 요청 SNS 광고 카피 5개를 만들어줘
출력형식 (Format) 원하는 응답 형태 표 형식, 30자 이내, 이모지 포함

SCORE 평가 프레임워크

본인이 쓴 프롬프트를 다음 5가지 기준으로 자기 평가하세요. 각 항목 20점 만점, 총 100점.

S
구체성 (Specificity)
요청이 모호하지 않고 명확한가?
C
맥락 (Context)
충분한 배경 정보를 제공했는가?
O
출력지정 (Output)
결과 형식을 명시했는가?
R
역할부여 (Role)
적절한 전문가 역할을 지정했는가?
E
효과성 (Effectiveness)
실제로 원하는 결과를 얻을 수 있는가?
점수 해석
  • 80점 이상 — 우수: 거의 그대로 써도 좋은 프롬프트
  • 60-79점 — 양호: 1-2가지 보강하면 우수
  • 40-59점 — 보통: 핵심 구성요소를 더 추가하세요
  • 40점 미만 — 개선 필요: 처음부터 다시 설계

좋은 vs 나쁜 프롬프트

① 자기소개서

나쁜 프롬프트 (S0 C0 O0 R0)
자기소개 써줘
좋은 프롬프트 (S5 C5 O4 R5 E5 = 24/25)
너는 IT 회사 신입 채용 담당자야.
한라대학교 컴공과 24세 지원자가
IT 스타트업에 보낼 자기소개서를 작성해줘.
1000자 이내, 진취적이지만 겸손한 톤.
강점 3가지를 항목으로 정리해줘.

② 코드 요청

나쁜 프롬프트
로그인 만들어줘
좋은 프롬프트
너는 React 시니어 개발자야.
HTML + CSS + JavaScript로
이메일 + 비밀번호 로그인 폼을 만들어줘.
- 빈 칸 검증
- 모바일 반응형
- 한국어 라벨
- 다크 블루 톤
파일명은 login.html, 한 파일에.

③ 학습 도움

나쁜 프롬프트
미분 설명해줘
좋은 프롬프트
너는 친절한 고등학교 수학 선생님이야.
나는 고등 수학을 거의 까먹은 비전공 대학교 1학년이야.
미분이 무엇인지 일상생활 예시 2가지로 설명해줘.
수식은 최소화. 마지막에 30초 안에 외울
한 줄 요약을 붙여줘.

시나리오 실습 5가지

각 시나리오에 대해 본인만의 프롬프트를 직접 작성해 보세요. Claude/ChatGPT에 던지고 결과를 비교.

① 마케팅 콘텐츠 작성

신제품 출시를 위한 SNS 홍보 문구를 작성하는 프롬프트를 만드세요.

힌트 — 제품 정보, 타깃 고객층, 톤앤매너, 채널(인스타/페북/X), 글자수 제한을 포함해 보세요.

② 코드 리뷰 요청

본인이 작성한 코드에 대한 리뷰를 AI에게 요청하는 프롬프트를 만드세요.

힌트 — 언어, 코드 목적, 리뷰 관점(성능/보안/가독성), 출력 형식(피드백 표/우선순위)을 포함.

③ 학습 자료 생성

특정 주제에 대한 교육 자료를 만드는 프롬프트를 작성하세요.

힌트 — 학습 대상(연령·수준), 분량, 형식(슬라이드/문서/퀴즈), 핵심 내용 영역을 포함.

④ 이메일 답장

받은 이메일에 답장하는 프롬프트를 작성하세요.

힌트 — 받는 사람, 본인 신분, 답장 톤(격식/친근), 핵심 메시지, 길이를 포함.

⑤ 발표자료 개요

특정 주제의 발표자료 개요를 만드는 프롬프트를 작성하세요.

힌트 — 발표 시간, 청중, 발표 목적, 구조(후크→본론→마무리), 슬라이드 수.

최종 체크리스트

실전에서 프롬프트를 던지기 전 마지막 점검:

  • AI에게 어떤 역할을 부여했는가? (Role)
  • 현재 상황·배경·대상을 충분히 설명했는가? (Context)
  • 구체적으로 무엇을 만들어 달라고 했는가? (Instruction)
  • 원하는 출력 형식·길이·톤을 명시했는가? (Format)
  • 모호한 표현("좋게", "괜찮게")을 구체화했는가?
  • 결과가 별로면 어디를 다듬을지 알 수 있는가?
  • 한 번에 너무 많은 것을 요구하지 않았는가?
  • 실제로 본인이 받았을 때 만족할 만한 답이 가능한가?
